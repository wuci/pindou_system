# 桌台布局配置存入数据库

**日期：** 2026-03-30
**修改人：** wuci

## 功能说明

将桌台布局配置从localStorage改为存储到数据库中，支持按分类分别保存布局配置。

## 修改内容

### 数据库

**新增表：** `biz_table_layout_config`

```sql
CREATE TABLE IF NOT EXISTS `biz_table_layout_config` (
    `id` BIGINT(20) NOT NULL AUTO_INCREMENT,
    `category_id` BIGINT(20) NOT NULL COMMENT '分类ID，0表示全局默认布局',
    `config` TEXT NOT NULL COMMENT '布局配置JSON',
    `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_category_id` (`category_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='桌台布局配置表';
```

### 后端

**新增实体类：** `TableLayoutConfig.java`

**新增DTO：**
- `TableLayoutConfigRequest.java` - 请求DTO
- `TableLayoutConfigResponse.java` - 响应DTO

**新增接口：**
- `GET /table-layout-config/{categoryId}` - 获取布局配置
- `POST /table-layout-config` - 保存布局配置

### 前端

**新增API：** `frontend/src/api/tableLayout.ts`

**修改组件：** `TableLayoutEditor.vue`
- 添加 `categoryId` 属性
- 修改 `loadLayout` 函数，从数据库加载布局
- 修改 `saveLayout` 函数，保存到数据库
- 修改 `resetLayout` 函数，清除数据库配置
- 监听 `categoryId` 变化，自动重新加载布局

## 使用说明

1. **编辑布局：**
   - 点击"布局视图" → "编辑布局"
   - 拖动桌台到想要的位置
   - 点击"保存布局"

2. **切换分类：**
   - 切换不同分类时，自动加载对应的布局配置
   - 每个分类的布局配置独立保存

3. **重置布局：**
   - 点击"重置"按钮恢复默认网格布局

## 布局配置格式

```json
[
  {
    "id": 1,
    "x": 20,
    "y": 20,
    "width": 120,
    "height": 120
  }
]
```

## 注意事项

1. 执行SQL文件创建数据库表
2. 布局配置按分类ID分别存储
3. categoryId=0 表示全局默认布局
4. 当桌台列表变化时，会自动重新加载布局
