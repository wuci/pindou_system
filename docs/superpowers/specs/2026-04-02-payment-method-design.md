# 支付方式功能 - 设计文档

**日期**: 2026-04-02
**作者**: wuci
**状态**: 设计中

## 一、功能概述

为桌台计时系统添加支付方式功能，支持线下支付、线上支付、会员余额支付和组合支付。

## 二、需求描述

### 2.1 支付方式定义

**非会员**（2种）：
- 线下支付 (offline)
- 线上支付 (online)

**会员**（4种）：
- 线下支付 (offline)
- 线上支付 (online)
- 会员余额 (balance)
- 组合支付 (combined) - 余额+线下

### 2.2 组合支付逻辑

**系统自动计算**：
1. 优先使用会员余额
2. 余额不足时，剩余部分使用线下支付
3. 开始计时不扣费，结账时扣除余额

**示例**：
- 费用：100元
- 余额：30元
- 结果：余额支付30元 + 线下支付70元

### 2.3 支付方式标记

- 线上/线下仅作为标记，实际支付行为都在店内完成
- 用于后续统计和分析

## 三、数据库设计

### 3.1 订单表修改

**表名**: `biz_order`

**新增字段**:
```sql
ALTER TABLE biz_order
ADD COLUMN payment_method VARCHAR(20) COMMENT '支付方式:offline-线下,online-线上,balance-余额,combined-组合',
ADD COLUMN balance_amount DECIMAL(10,2) DEFAULT 0 COMMENT '余额支付金额',
ADD COLUMN other_payment_amount DECIMAL(10,2) DEFAULT 0 COMMENT '其他方式支付金额';
```

**字段说明**:
- `payment_method`: 支付方式枚举值
- `balance_amount`: 余额支付金额（组合支付时使用）
- `other_payment_amount`: 其他方式支付金额（组合支付时使用）

## 四、业务流程

### 4.1 开始计时流程

1. 用户选择桌台
2. 用户选择是否会员
3. 如果是会员，选择会员信息
4. 根据是否会员显示对应支付方式
5. 用户选择支付方式
6. 提交开始计时请求，**不扣费**，仅记录支付方式

### 4.2 结账流程

1. 系统计算最终费用
2. 检查订单支付方式：
   - `offline` / `online`: 不扣除余额
   - `balance`: 检查余额是否充足
     - 充足 → 扣除全部余额
     - 不足 → 提示余额不足
   - `combined`: 计算余额使用量
     - 余额 >= 费用 → 全部余额支付
     - 余额 < 费用 → 余额全部使用 + 剩余线下
3. 扣除会员余额（如适用）
4. 更新订单支付状态
5. 记录消费记录（仅余额支付部分）

## 六、技术实现要点

### 6.1 后端

1. **实体类修改**
   - `Order.java`: 添加支付方式相关字段

2. **DTO修改**
   - `StartTimerRequest.java`: 添加 `paymentMethod` 字段
   - `TableInfoResponse.java`: 返回支付方式信息

3. **Service层修改**
   - `TableService.startTimer()`: 保存支付方式
   - `TableService.endTimer()`: 处理余额扣除

4. **余额扣除逻辑**
   - 事务处理确保余额扣减原子性
   - 记录消费记录
   - 更新会员累计消费金额

### 6.2 前端

1. **API层**
   - 添加支付方式枚举定义
   - 修改开始计时API参数

2. **组件修改**
   - `StartTimerDialog.vue`: 添加支付方式选择
   - `BillDialog.vue`: 显示支付明细

3. **类型定义**
   - TypeScript 接口定义

## 七、异常处理

1. **余额不足**：提示用户充值或选择其他支付方式
2. **余额扣除失败**：回滚事务，提示用户
3. **会员信息变更**：实时刷新会员余额信息
