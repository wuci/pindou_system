# Token不匹配异常处理优化

**日期：** 2026-03-30
**修改人：** wuci

## 问题描述

当Redis中的Token不匹配时（如用户在其他地方登录导致原Token失效），后端只清除认证上下文但不返回401状态码，导致前端无法正确跳转到登录页。

## 修改内容

### 修改文件：`backend/pdsystem-server/src/main/java/com/pindou/timer/filter/JwtAuthenticationFilter.java`

#### 主要变更：

1. **添加ObjectMapper依赖**：用于JSON序列化错误响应
2. **修改Token验证失败处理逻辑**：
   - 当`validateToken`返回false时，立即返回401状态码
   - 不再继续执行过滤器链
3. **添加异常捕获**：处理Token验证过程中的异常
4. **新增`handleUnauthorized`方法**：统一处理未授权响应

#### 关键代码变更：

**修改前：**
```java
if (userService.validateToken(token)) {
    // 设置认证信息
}
} catch (Exception e) {
    SecurityContextHolder.clearContext();
}
filterChain.doFilter(request, response);
```

**修改后：**
```java
if (userService.validateToken(token)) {
    // 设置认证信息
} else {
    handleUnauthorized(response, "登录已过期，请重新登录");
    return;
}
} catch (Exception e) {
    handleUnauthorized(response, "Token验证失败");
    return;
}
filterChain.doFilter(request, response);
```

**新增方法：**
```java
private void handleUnauthorized(HttpServletResponse response, String message) throws IOException {
    SecurityContextHolder.clearContext();
    response.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
    response.setContentType("application/json;charset=UTF-8");
    Result<Void> result = Result.error(401, message);
    response.getWriter().write(objectMapper.writeValueAsString(result));
    response.getWriter().flush();
}
```

## 效果

- Redis Token不匹配时，立即返回401状态码
- 前端接收到401后，显示提示并跳转到登录页
- 清除本地所有存储，确保安全

## 测试场景

1. 用户在A设备登录
2. 用户在B设备登录（Redis中的Token更新）
3. A设备发起请求
4. 后端检测到Token不匹配，返回401
5. 前端跳转到登录页
