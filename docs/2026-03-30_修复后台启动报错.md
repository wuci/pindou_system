# 修复后台启动报错

## 问题描述
后台启动报错，编译失败。

## 问题原因
`BatchDeleteTableRequest` DTO 在 `pdsystem-dto` 模块中使用了 `javax.validation` 注解（如 `@NotEmpty`），但该模块的 pom.xml 中缺少 `spring-boot-starter-validation` 依赖，导致编译失败。

## 解决方案
在 `pdsystem-dto/pom.xml` 中添加缺失的依赖：
- `spring-boot-starter-validation` - 提供 JSR-303 验证注解支持
- `knife4j-openapi3-spring-boot-starter` - 提供 Swagger 注解支持

## 修改文件
- 修改文件：backend/pdsystem-dto/pom.xml
