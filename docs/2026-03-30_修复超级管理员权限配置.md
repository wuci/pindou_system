# 修复超级管理员权限配置

## 问题描述
admin账号登录后看不到角色管理和系统设置菜单

## 根本原因
数据库中超级管理员角色的权限不是通配符`["*"]`，而是具体的权限列表：
- 后台返回：`["user:view","user:create","user:update","user:delete","user:resetPassword","role:view","role:create","role:update","role:delete","table:view","table:create","table:update","table:delete","order:view","order:create","order:update","order:delete","order:export","config:view","config:update","statistics:view"]`
- 前端期望：`role:manage`和`system:config`

由于admin账号没有通配符权限，导致菜单不显示

## 解决方案
通过API更新超级管理员角色权限为`["*"]`：
```bash
PUT /api/roles
{
  "id": "a0000000-0000-0000-0000-000000000001",
  "permissions": ["*"]
}
```

## 验证
更新后admin账号登录返回：
```json
{
  "userInfo": {
    "permissions": ["*"]
  }
}
```

## 后续操作
用户需要退出登录后重新登录，以刷新权限缓存
