# 修复角色管理和系统设置菜单权限

## 问题描述
角色管理和系统设置菜单无法显示，原因是：
1. **角色管理**菜单需要`role:manage`权限，但"店长"角色没有此权限
2. **系统设置**菜单需要`system:config`权限

## 解决方案
为"店长"角色添加`role:manage`权限

## 修改文件
1. **backend/sql/data.sql** - 更新初始化数据脚本，为店长角色添加`role:manage`权限
2. **backend/sql/update_role_permissions.sql** - 创建更新脚本，用于更新现有数据库

## 执行更新
请手动执行以下SQL脚本来更新现有数据库：
```bash
mysql -uroot -p123456 pindou_timer < backend/sql/update_role_permissions.sql
```

或在MySQL客户端中执行：
```sql
USE `pindou_timer`;

UPDATE `sys_role`
SET `permissions` = '["table:view","table:start","table:pause","table:end","table:config","order:view","order:detail","order:export","report:view","system:config","user:view","user:create","user:edit","role:manage","log:view","remind:ignore"]',
    `updated_at` = UNIX_TIMESTAMP() * 1000
WHERE `code` = 'manager';
```

## 功能说明
- 超级管理员：拥有所有权限（*）
- 店长：新增角色管理权限，可以查看和编辑角色
- 系统设置菜单需要system:config权限，店长已有此权限

## 更新后操作
更新数据库后，需要：
1. 重新登录系统（刷新用户权限缓存）
2. 角色管理和系统设置菜单将正常显示
