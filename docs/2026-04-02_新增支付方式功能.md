# 新增支付方式功能

**日期**: 2026-04-02
**开发者**: wuci
**功能描述**: 新增桌台开始计时时的支付方式选择功能，支持线下支付、线上支付、会员余额支付和组合支付

## 功能概述

本次更新新增了桌台开始计时时的支付方式选择功能：
- **非会员**: 支持线下支付、线上支付
- **会员**: 支持线下支付、线上支付、会员余额、组合支付（余额+线下）
- 组合支付自动计算：优先使用会员余额，不足部分使用线下支付
- 余额扣除在结账时进行，不在开始计时时扣除
- 线上/线下仅作为标记，实际支付行为在店内完成

#### 修改结账逻辑（endTimer方法）
添加余额扣除逻辑：
- 检查支付方式是否为 `balance` 或 `combined`
- 对于 `balance` 方式：验证余额是否充足，扣除全部金额
- 对于 `combined` 方式：使用全部余额，剩余部分计入 `otherPaymentAmount`
- 余额不足时抛出异常

```java

    // 余额扣除逻辑...
}
```

## 技术要点

1. **时间戳统一**: 所有时间字段使用毫秒时间戳，确保前后端一致
2. **余额扣除时机**: 在结账时扣除余额，不在开始计时时扣除
3. **组合支付计算**: 自动计算余额和其他支付方式的金额分配
4. **事务安全**: 余额扣除使用事务确保数据一致性
5. **用户体验**: 根据是否选择会员动态显示可用的支付方式

## 测试要点

1. 非会员开始计时：只显示线下、线上两种支付方式
2. 会员开始计时：显示全部四种支付方式
3. 选择组合支付时显示余额和线下支付的金额分配
4. 结账时验证余额扣除是否正确
5. 余额不足时是否正确抛出异常
