## 功能特性

### 折扣类型
1. **固定折扣**：适用于所有订单，按设置的折扣率计算
2. **会员折扣**：仅适用于指定会员等级的会员
3. **活动折扣**：可用于特定活动，可设置有效期

### 折扣设置
- 折扣率：0.1-1.0（0.9表示9折）
- 最低消费金额：可设置，满足条件才可享受折扣
- 最高优惠金额：可设置，限制最大优惠金额
- 会员等级：会员折扣类型必须指定会员等级
- 有效期：可设置开始时间和结束时间
- 状态：启用/禁用
- 排序：数值越小越靠前

### API接口
- `GET /api/discounts` - 获取折扣列表（分页）
- `GET /api/discounts/active` - 获取所有启用的折扣
- `POST /api/discounts` - 创建折扣
- `PUT /api/discounts/{id}` - 更新折扣
- `DELETE /api/discounts/{id}` - 删除折扣
- `PUT /api/discounts/{id}/status` - 更新折扣状态
- `POST /api/discounts/calculate` - 计算订单折扣

### 权限
- `system:discount` - 折扣设置权限（已添加到 sys_permission 表）

## 数据库表结构

### biz_discount
| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | 折扣ID（UUID） |
| name | VARCHAR(50) | 折扣名称 |
| type | TINYINT(1) | 折扣类型：1-固定折扣 2-会员折扣 3-活动折扣 |
| discount_rate | DECIMAL(4,3) | 折扣率（0.9表示9折） |
| min_amount | DECIMAL(10,2) | 最低消费金额 |
| max_discount | DECIMAL(10,2) | 最高优惠金额 |
| member_level_id | BIGINT | 会员等级ID |
| start_time | BIGINT | 开始时间（毫秒时间戳） |
| end_time | BIGINT | 结束时间（毫秒时间戳） |
| status | TINYINT(1) | 状态：0-禁用，1-启用 |
| sort | INT | 排序 |
| description | VARCHAR(200) | 描述 |
| created_at | BIGINT | 创建时间 |
| updated_at | BIGINT | 更新时间 |
| deleted_at | BIGINT | 删除标记 |
