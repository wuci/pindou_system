# 2026-04-06 Redis折扣记录优化

## 修改描述
使用Redis存储订单折扣记录，优化多次折扣的平均值计算逻辑。订单完成后自动清理Redis数据。

## 业务逻辑说明

### Redis数据结构
- Key格式：`order:discount:{orderId}`
- 数据类型：List
- 存储内容：折扣率列表（BigDecimal）
- 过期时间：24小时

### 平均折扣率计算流程
1. **开始计时**：选择折扣后，将折扣率存入Redis
2. **续费**：
   - 从Redis获取已有折扣率列表
   - 计算平均值 = (已有折扣率总和 + 新折扣率) / (已有数量 + 1)
   - 将新折扣率追加到Redis List
   - 最终金额 = 原价 × 平均折扣率
3. **结账完成**：删除Redis中的折扣记录

### 示例场景
1. 用户开始计时，选择7折活动折扣
   - Redis: `[0.7]`
   - 平均折扣率: 0.7
   - 最终价格 = 原价 × 0.7

2. 用户续费，选择8折活动折扣
   - Redis: `[0.7, 0.8]`
   - 平均折扣率: (0.7 + 0.8) / 2 = 0.75
   - 最终价格 = 原价 × 0.75

3. 用户续费，选择9折活动折扣
   - Redis: `[0.7, 0.8, 0.9]`
   - 平均折扣率: (0.7 + 0.8 + 0.9) / 3 = 0.8
   - 最终价格 = 原价 × 0.8

4. 订单完成后，清除Redis数据

## 优化点
1. **数据结构更清晰**：使用Redis List天然支持多次折扣记录
2. **计算逻辑简化**：直接从Redis获取列表计算平均值
3. **自动清理**：订单完成后自动清除，避免数据累积
4. **性能优化**：Redis操作比JSON字符串拼接更高效
5. **数据库整洁**：不需要在Order表中存储复杂的JSON数据
