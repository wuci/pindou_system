# Token滑动过期优化

## 修改时间
2026-04-06

## 修改内容

### 问题背景
- 原 Token 有效期为 7 天，服务重启后仍然保持登录状态
- 没有滑动过期机制，用户无操作也不会自动退出

### 解决方案
1. **修改 Token 过期时间**：从 7 天改为 30 分钟
2. **实现滑动过期机制**：每次验证 Token 成功后刷新过期时间

### 修改文件
- `backend/pdsystem-service/src/main/java/com/pindou/timer/service/impl/UserServiceImpl.java`

### 具体修改

#### 1. 修改常量定义（第 45-50 行）
```java
private static final String TOKEN_PREFIX = "token:";
/**
 * Token过期时间（30分钟，单位：秒）
 * 用户30分钟内无操作将自动退出登录
 */
private static final long TOKEN_EXPIRE_TIME = 30 * 60;
```

#### 2. 增强 validateToken 方法（第 166-197 行）
- 在 Token 验证成功后，调用 `redisUtil.set()` 刷新过期时间
- 实现滑动过期效果

### 效果
- 用户持续操作时，Token 一直保持有效
- 用户 30 分钟无操作后，Token 自动失效
- 服务重启后，Redis 中的 Token 数据会清空，需要重新登录
