# 2026-04-03 权限显示问题排查指南

## 问题描述

给店长角色配置了对应的功能，但界面只展示了工作台、订单管理、数据统计，其他的都没展示。

## 重要提示：修改权限后必须重新登录

**⚠️ 最常见的问题**：修改角色权限后，该角色的用户没有退出登录，导致使用的是旧的权限数据。

**解决方案**：
1. 修改角色权限后
2. **该角色的所有用户必须退出登录**
3. **重新登录** 才能看到新的权限

用户的权限信息在登录时加载并保存在浏览器 localStorage 中，修改角色权限不会自动更新已登录用户的权限。

## 排查步骤

### 1. 确认是否重新登录

修改角色权限后，**必须退出登录并重新登录**才能生效。

如果修改权限后没有重新登录，在浏览器控制台看到的权限列表是旧的。

### 2. 查看当前登录用户的权限

在浏览器控制台执行以下代码，查看当前用户的权限列表：

```javascript
// 方法一：直接查看
import { useUserStore } from '@/stores/user'
const userStore = useUserStore()
console.log('用户权限列表:', userStore.permissions)

// 方法二：检查特定权限
console.log('是否有 table:view:', userStore.hasPermission('table:view'))
console.log('是否有 member:view:', userStore.hasPermission('member:view'))
```

### 3. 查看数据库中的角色权限

连接到数据库，执行以下查询：

```sql
SELECT
    id,
    name,
    code,
    JSON_PRETTY(permissions) as permissions,
    status
FROM sys_role
WHERE code = 'role_manager';
```

### 4. 对比权限列表

对比数据库中的权限列表和浏览器控制台显示的权限列表：
- 如果数据库中的权限是完整的（包含 table:view, member:view 等）
- 但浏览器控制台显示的权限是旧的
- 说明**没有重新登录**

### 5. 清除浏览器缓存并重新登录

如果确认权限配置正确，但前端仍然显示不正确：

1. 退出登录
2. 清除浏览器 localStorage：
   ```javascript
   // 在浏览器控制台执行
   localStorage.clear()
   sessionStorage.clear()
   ```
3. 重新登录

## 正确的操作流程

### 修改角色权限的完整流程

1. **以管理员身份登录**
2. 进入"角色管理"页面
3. 点击角色的"编辑"按钮
4. 在权限配置树中勾选需要的权限：
   - 可以勾选模块节点（会自动选中所有子节点）
   - 也可以精确勾选具体的操作权限
5. 点击"确定"保存
6. **通知该角色的所有用户退出登录并重新登录**

### 用户重新登录后的验证

1. 重新登录
2. 检查侧边栏菜单是否正确显示
3. 检查各个功能页面的按钮是否正确显示
4. 在浏览器控制台验证权限列表

## 常见问题

### Q1: 为什么修改权限后侧边栏菜单没变化？

**A**: 没有退出登录并重新登录。用户的权限在登录时加载，修改权限后需要重新登录。

### Q2: 为什么权限树勾选后保存成功，但数据库中没有权限？

**A**: 这是另一个问题，已通过修改 `getCheckedKeys(false)` 修复。确保使用的是最新的前端代码。

### Q3: 为什么只显示了部分菜单？

**A**: 检查是否正确配置了模块权限。例如，要显示"桌台管理"菜单，需要勾选 `table:view` 权限。

### Q4: 为什么菜单显示了，但操作按钮不显示？

**A**: 除了模块权限（控制菜单显示），还需要配置操作权限（控制按钮显示）。例如：
- 要显示"开始计时"按钮，需要勾选 `table:start` 权限
- 要显示"新增会员"按钮，需要勾选 `member:create` 权限
