# 建表脚本字段补充修复

## 日期
2026-04-04

## 问题描述
检查发现 `01_建表脚本.sql` 中缺少部分字段，与现有的增量 SQL 脚本不一致。

## 缺失字段统计

### biz_order 表缺失字段
| 字段名 | 类型 | 说明 | 来源文件 |
|--------|------|------|----------|
| channel | VARCHAR(50) | 订餐渠道：store-门店,phone-电话,online-线上 | add_order_channel.sql |
| original_amount | DECIMAL(10,2) | 原价（折扣前） | update_order_member.sql |

### biz_order 表字段修正
| 字段名 | 修改前 | 修改后 | 说明 |
|--------|--------|--------|------|
| member_discount | 默认值 1.00 | 默认值 0.00 | 修正默认值 |

### 外键约束移除
- 移除 `biz_member` 表的 `fk_member_level` 外键约束
- 移除 `biz_recharge_record` 表的 `fk_recharge_member` 外键约束
- 移除 `biz_consumption_record` 表的 `fk_consumption_member` 外键约束
- 移除 `sys_user` 表的 `fk_user_role` 外键约束

**原因**：外键约束在数据迁移和删除操作时可能造成麻烦，通过应用层保证数据一致性更加灵活。

## 修改内容

### 文件：backend/sql/20260403/01_建表脚本.sql

1. **biz_order 表添加缺失字段**
```sql
-- 在 preset_duration 后添加
`channel` VARCHAR(50) DEFAULT 'store' COMMENT '订餐渠道：store-门店,phone-电话,online-线上',

-- 在 amount 后添加
`original_amount` DECIMAL(10,2) DEFAULT 0.00 COMMENT '原价（折扣前）',

-- 修正 member_discount 默认值
`member_discount` DECIMAL(10,2) DEFAULT 0.00 COMMENT '会员折扣金额',

-- 添加索引
KEY `idx_channel` (`channel`)
```

2. **移除所有外键约束**
- 删除建表语句中的 `CONSTRAINT` 外键定义
- 删除脚本末尾的 `ALTER TABLE ... ADD CONSTRAINT ... FOREIGN KEY` 语句
