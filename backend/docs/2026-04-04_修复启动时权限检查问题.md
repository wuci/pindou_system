# 修复启动时权限检查问题

## 日期
2026-04-04

## 问题描述
1. 用户刚启动应用时，如果 localStorage 中存在 token 和 userInfo，但 permissions 为空或缺少权限，会报错"无权限，重定向到首页。需要权限: dashboard:view 用户权限"，而不是直接跳转到登录页。
2. 用户权限中没有 `dashboard:view` 权限时，会无限循环跳转
3. 登录界面无法显示，一直报错"无权限访问"

## 问题原因
1. 用户启动时，从 localStorage 恢复了 token 和 userInfo
2. `isLogin` 返回 true（因为有 token 和 userInfo）
3. 但 `permissions` 可能为空或没有 `dashboard:view` 权限
4. 路由守卫判断已登录后进行权限检查，失败后固定跳转到 `/dashboard`
5. `/dashboard` 也需要 `dashboard:view` 权限，造成无限循环
6. `getUserInfo()` 调用失败时可能与 `request.ts` 的 401 处理冲突

## 修改内容

### 文件：frontend/src/router/index.ts

1. **新增 `getFirstAccessibleRoute` 函数**：根据用户权限获取第一个有权访问的路由路径

2. **路由守卫改为异步函数**：将 `beforeEach` 改为 `async` 函数

3. **添加防跳转标志**：使用 `isRedirecting` 标志防止无限循环跳转

4. **登录页直接放行**：登录页不进行任何额外检查

5. **权限为空时清除登录状态**：不再调用 API 获取用户信息，直接清除本地数据并跳转登录页

6. **权限检查失败时动态跳转**：无权限时跳转到第一个有权访问的页面

7. **根路径动态重定向**：访问根路径时，跳转到第一个有权访问的页面

### 文件：frontend/src/stores/user.ts

1. **新增 `clearAuth()` 方法**：清除本地认证数据而不调用后端 API
2. **修改 `getUserInfo()` 方法**：移除内部的 `this.logout()` 调用，由调用方处理错误

### 文件：frontend/src/views/NotFound/index.vue（新建）

创建404页面组件，支持返回上一页和返回首页功能

## 修改后的流程

```
启动应用
  ↓
路由守卫检查（检查 isRedirecting 标志）
  ↓
是否登录页？→ 是 → 放行
  ↓
是否根路径？→ 是 → 已登录？→ 是 → 权限为空？→ 是 → 清除数据 → 跳转登录页
  ↓                    ↓                   ↓
  否                  否                  否 → 获取第一个有权访问页面 → 跳转
  ↓                    ↓
  否                  否 → 跳转登录页
  ↓
需要登录？→ 否 → 放行
  ↓
是 → 已登录？→ 否 → 跳转登录页
  ↓
是 → 权限为空？→ 是 → 清除数据 → 跳转登录页
  ↓
否 → 检查路由权限 → 有权限 → 放行
  ↓
无权限 → 获取第一个有权访问页面 → 有 → 跳转该页面
                                      ↓
                                    无 → 清除数据 → 跳转登录页
```

## 代码变更

```typescript
// 新增函数
const getFirstAccessibleRoute = (permissions: string[]): string => {
  // 超级管理员直接返回工作台
  if (permissions.includes('*')) {
    return '/dashboard'
  }

  // 按优先级查找第一个有权访问的路由
  const routeOrder = [
    { path: '/dashboard', permission: 'dashboard:view' },
    { path: '/tables', permission: 'table:view' },
    { path: '/orders', permission: 'order:view' },
    { path: '/reports', permission: 'statistics:view' },
    { path: '/users', permission: 'user:view' },
    { path: '/roles', permission: 'role:view' },
    { path: '/logs', permission: 'log:view' },
    { path: '/members', permission: 'member:view' },
    { path: '/member-levels', permission: 'member:level' },
    { path: '/settings', permission: 'system:view' }
  ]

  for (const route of routeOrder) {
    if (checkModulePermission(permissions, route.permission)) {
      return route.path
    }
  }
  return ''
}

// 防止无限跳转
let isRedirecting = false

// 路由守卫
router.beforeEach(async (to, from, next) => {
  // 检查是否正在重定向
  if (isRedirecting) {
    next()
    return
  }

  // 登录页直接放行
  if (to.path === '/login') {
    next()
    return
  }

  // 权限为空时清除登录状态（不调用 API）
  if (to.meta.requiresAuth !== false && userStore.isLogin && (!userStore.permissions || userStore.permissions.length === 0)) {
    userStore.clearAuth()
    isRedirecting = true
    next({ path: '/login' })
    setTimeout(() => { isRedirecting = false }, 100)
    return
  }

  // ... 其他逻辑
})

// user store 新增方法
clearAuth() {
  this.token = null
  this.userInfo = null
  this.permissions = []
  storage.remove('token')
  storage.remove('userInfo')
}
```
