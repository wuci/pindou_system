# 路由守卫权限判断逻辑优化

**日期：** 2026-04-03
**修改人：** wuci
**问题描述：** 店长用户虽然侧边栏菜单显示正常，但点击菜单时提示"您没有访问该页面的权限"

## 问题分析

### 原有路由守卫逻辑（存在问题）

```javascript
// 检查权限（只有已登录用户才检查权限）
if (to.meta.permission) {
  if (!userStore.hasPermission(to.meta.permission as string)) {
    ElMessage.warning('您没有访问该页面的权限')
    next({ path: '/dashboard' })
    return
  }
}
```

### 路由配置与用户权限不匹配

| 路由 | 路由要求权限 | 用户实际权限 | 匹配情况 |
|-----|------------|-------------|---------|
| /tables | `table:view` | `table:category`、`table:config` 等 | ❌ 不匹配 |
| /members | `member:view` | `member:create`、`member:update` 等 | ❌ 不匹配 |
| /reports | `report:view` | `statistics:view` | ❌ 不匹配 |
| /settings | `system:config` | `system:rule` | ❌ 不匹配 |

### 问题结果

用户虽然能看到菜单，但点击时路由守卫严格检查特定权限，导致无法访问。

## 解决方案

### 1. 优化路由守卫逻辑

添加 `checkModulePermission` 函数，支持模块级权限检查：

```javascript
/**
 * 检查是否有指定模块下的任意权限
 * @param permissions 用户权限列表
 * @param permission 路由要求的权限
 * @returns 是否有权限
 */
const checkModulePermission = (permissions: string[], permission: string): boolean => {
  // 超级管理员
  if (permissions.includes('*')) {
    return true
  }

  // 先检查精确匹配
  if (permissions.includes(permission)) {
    return true
  }

  // 如果要求的权限是 xxx:view 格式，检查是否有该模块下的任意权限
  // 例如：table:view -> 检查是否有 table: 开头的任意权限
  if (permission.endsWith(':view')) {
    const prefix = permission.substring(0, permission.lastIndexOf(':') + 1)
    return permissions.some(p => p.startsWith(prefix))
  }

  return false
}

// 使用新函数检查权限
if (to.meta.permission) {
  const requiredPermission = to.meta.permission as string
  if (!checkModulePermission(userStore.permissions, requiredPermission)) {
    ElMessage.warning('您没有访问该页面的权限')
    next({ path: '/dashboard' })
    return
  }
}
```

### 2. 修正路由权限配置

| 路由 | 修改前 | 修改后 | 原因 |
|-----|--------|--------|------|
| /reports | `report:view` | `statistics:view` | 与用户权限一致 |
| /settings | `system:config` | `system:view` | 支持模块级权限检查 |

### 优化后效果

| 路由 | 路由要求 | 用户权限 | 匹配逻辑 | 结果 |
|-----|---------|---------|---------|------|
| /tables | `table:view` | `table:category` 等 | 模块级匹配 | ✅ 通过 |
| /members | `member:view` | `member:create` 等 | 模块级匹配 | ✅ 通过 |
| /reports | `statistics:view` | `statistics:view` | 精确匹配 | ✅ 通过 |
| /settings | `system:view` | `system:rule` | 模块级匹配 | ✅ 通过 |

## 影响范围

- 影响所有需要权限控制的路由访问
- 用户只要拥有某个模块下的任意权限，即可访问该模块路由
- 页面内的按钮级别权限控制在各自组件中继续生效

## 权限判断逻辑说明

### 精确匹配优先
如果用户拥有路由要求的精确权限，直接放行。

### 模块级匹配
如果路由要求的是 `xxx:view` 权限，但用户没有精确权限，则检查用户是否有该模块下的任意权限：
- 路由要求：`table:view`
- 用户权限：`table:category`、`table:config`、`table:start` 等
- 判断逻辑：有任意 `table:` 开头的权限即可通过

### 不影响其他权限
对于非 `:view` 后缀的权限（如 `system:rule`、`role:manage`），继续使用精确匹配。
