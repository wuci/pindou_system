# 角色管理删除按钮逻辑优化

**日期：** 2026-04-03
**修改人：** wuci
**问题描述：** 管理员账号登录后，角色管理页面的删除按钮都是禁用的，无法删除角色

## 原有逻辑（存在问题）

```vue
<!-- 操作列 -->
<el-button
  v-if="permissions.canDelete"
  type="danger"
  size="small"
  link
  @click="handleDelete(row)"
  :disabled="row.isBuiltIn === 1"  <!-- 系统内置角色禁用删除按钮 -->
>
  删除
</el-button>

<!-- 删除函数 -->
const handleDelete = async (row: RoleInfo) => {
  if (row.isBuiltIn === 1) {  <!-- 系统内置角色不允许删除 -->
    ElMessage.warning('系统内置角色不允许删除')
    return
  }
  // ... 删除逻辑
}
```

### 问题分析

1. **所有系统内置角色（`isBuiltIn === 1`）的删除按钮都被禁用**
2. 包括"店长"、"店员"等系统预置角色都无法删除
3. 实际上应该只有超级管理员角色不可删除，其他角色都应该可以删除

## 优化后的逻辑

### 1. 操作列显示逻辑

```vue
<el-table-column label="操作" width="180" fixed="right">
  <template #default="{ row }">
    <el-button v-if="permissions.canUpdate" type="primary" size="small" link @click="handleEdit(row)">
      编辑
    </el-button>
    <!-- 只有超级管理员角色（code === 'super_admin'）不显示删除按钮 -->
    <el-button
      v-if="permissions.canDelete && row.code !== 'super_admin'"
      type="danger"
      size="small"
      link
      @click="handleDelete(row)"
    >
      删除
    </el-button>
  </template>
</el-table-column>
```

### 2. 删除函数逻辑

```javascript
// 删除 - 移除了系统内置角色的限制
const handleDelete = async (row: RoleInfo) => {
  try {
    await ElMessageBox.confirm(
      `确认要删除角色 "${row.name}" 吗？删除后不可恢复！`,
      '删除角色',
      {
        confirmButtonText: '确认',
        cancelButtonText: '取消',
        type: 'warning'
      }
    )

    await deleteRole(row.id)
    ElMessage.success('删除成功')
    loadRoles()
  } catch (error) {
    if (error !== 'cancel') {
      ElMessage.error('删除失败')
    }
  }
}
```

## 优化后效果

| 角色 | 角色编码 | 是否内置 | 修改前 | 修改后 |
|-----|---------|---------|--------|--------|
| 超级管理员 | `super_admin` | 是 | ❌ 按钮禁用 | ❌ 不显示删除按钮 |
| 店长 | `manager` | 是 | ❌ 按钮禁用 | ✅ 可删除 |
| 店员 | `staff` | 是 | ❌ 按钮禁用 | ✅ 可删除 |
| 只读用户 | `readonly` | 是 | ❌ 按钮禁用 | ✅ 可删除 |
| 自定义角色 | 自定义 | 否 | ✅ 可删除 | ✅ 可删除 |

## 实现逻辑说明

### 显示逻辑
- 如果用户有 `role:delete` 权限 **且** 角色编码不是 `super_admin`，显示删除按钮
- 超级管理员角色的操作列中不显示删除按钮（使用 `v-if` 条件）

### 删除逻辑
- 移除了基于 `isBuiltIn` 的判断
- 只有能点击删除按钮的角色都可以删除
- 超级管理员角色根本看不到删除按钮，因此不会被误删

## 安全性考虑

1. **前端限制**：超级管理员角色的删除按钮不显示
2. **后端验证**：建议后端也要同步添加 `super_admin` 角色的删除限制
3. **数据库保护**：可以考虑在数据库层面添加约束或触发器

## 验证步骤

1. 使用管理员账号登录
2. 进入角色管理页面
3. 验证：
   - 超级管理员角色的操作列只有"编辑"按钮，没有"删除"按钮
   - 其他角色（店长、店员等）的"删除"按钮正常显示且可点击
   - 点击删除按钮，确认后可以成功删除角色
