# 2026-04-03 操作日志模板字符串解析修复

## 修改日期
2026-04-03

## 问题描述
操作日志中记录的内容显示为字面量模板字符串，而不是实际的参数值。
例如：`桌台【#tableId】结束计时并结账，会员ID【#request?.memberId】`

## 问题原因

### 原始 `parseSpEL` 方法的问题

原来的 `parseSpEL` 方法将整个描述字符串作为单个 SpEL 表达式解析：

```java
Expression expression = parser.parseExpression(spel);
Object value = expression.getValue(context);
```

当描述字符串是混合文本时，例如：
- `桌台【#tableId】结束计时并结账，会员ID【#request?.memberId】`
- `创建会员【#request.name】，手机号【#request.phone】`

SpEL 解析器会将整个字符串当作一个表达式来解析，导致解析失败，然后返回原始字符串。

### SpEL 表达式的两种使用方式

1. **纯表达式**：直接是一个表达式，如 `#tableId`
   - `parseExpression("#tableId")` → 返回 tableId 的值

2. **混合文本**：表达式嵌入在普通文本中，如 `桌台【#tableId】结束计时`
   - 需要使用模板解析或手动替换占位符

## 修复方案

### 修改的文件
`backend/pdsystem-server/src/main/java/com/pindou/timer/aspect/LogAspect.java`

### 修改内容（第 221-268 行）

**修复后的 `parseSpEL` 方法**：

```java
/**
 * 解析 SpEL 表达式
 * 支持模板式占位符，例如：桌台【#tableId】结束计时
 */
private String parseSpEL(String spel, ProceedingJoinPoint point, Class<?> targetClass) {
    if (StringUtils.isBlank(spel)) {
        return "";
    }

    // 如果不包含 SpEL 表达式标记（# 或 $），直接返回原字符串
    if (!spel.contains("#") && !spel.contains("$")) {
        return spel;
    }

    try {
        MethodSignature signature = (MethodSignature) point.getSignature();
        Method method = signature.getMethod();

        // 获取目标类的方法
        Method targetMethod = targetClass.getMethod(method.getName(), method.getParameterTypes());

        // 获取方法参数名和值
        String[] parameterNames = nameDiscoverer.getParameterNames(targetMethod);
        Object[] args = point.getArgs();

        // 创建 SpEL 上下文
        EvaluationContext context = new StandardEvaluationContext();

        // 设置参数到上下文
        if (parameterNames != null) {
            for (int i = 0; i < parameterNames.length; i++) {
                context.setVariable(parameterNames[i], args[i]);
            }
        }

        // 尝试解析整个表达式（兼容纯表达式情况）
        try {
            Expression expression = parser.parseExpression(spel);
            Object value = expression.getValue(context);
            if (value != null) {
                return value.toString();
            }
        } catch (Exception e) {
            // 整个字符串解析失败，尝试模板式替换
            log.debug("整体解析失败，尝试模板式替换：{}", spel);
        }

        // 模板式替换：查找所有 #xxx 或 #xxx?.yyy 模式并替换
        String result = spel;
        java.util.regex.Pattern pattern = java.util.regex.Pattern.compile("#(\\w+(?:\\?\\.\\w+)*)(?=[\\]】\\s,，。]|$)");
        java.util.regex.Matcher matcher = pattern.matcher(result);
        StringBuffer sb = new StringBuffer();

        while (matcher.find()) {
            String varExpression = matcher.group(0); // 完整匹配，如 #tableId 或 #request?.memberId
            try {
                Expression expression = parser.parseExpression(varExpression);
                Object value = expression.getValue(context);
                String replacement = value != null ? value.toString() : "";
                matcher.appendReplacement(sb, java.util.regex.Matcher.quoteReplacement(replacement));
            } catch (Exception ex) {
                log.debug("解析表达式 {} 失败，保持原样", varExpression);
                matcher.appendReplacement(sb, java.util.regex.Matcher.quoteReplacement(varExpression));
            }
        }
        matcher.appendTail(sb);

        return sb.toString();

    } catch (Exception e) {
        log.warn("解析 SpEL 表达式失败：{}", spel, e);
    }

    // 解析失败，返回原始字符串
    return spel;
}
```

### 修复要点

1. **两层解析策略**：
   - 首先尝试将整个字符串作为纯 SpEL 表达式解析（向后兼容）
   - 如果失败，则使用正则表达式提取占位符逐个替换

2. **正则表达式模式**：`#(\\w+(?:\\?\\.\\w+)*)(?=[\\]】\\s,，。]|$)`
   - `#(\\w+(?:\\?\\.\\w+)*)` - 匹配 `#tableId` 或 `#request?.memberId` 格式
   - `(?=[\\]】\\s,，。]|$)` - 后瞻匹配，确保占位符后是 `]`、`】`、空格、逗号或字符串结尾

3. **安全的占位符替换**：
   - 使用 `Matcher.quoteReplacement()` 避免替换字符串中的特殊字符被当作正则表达式
   - 单个表达式解析失败时保持原样，不影响其他占位符

## 支持的占位符格式

修复后支持以下占位符格式：

| 格式 | 示例 | 说明 |
|------|------|------|
| `#参数名` | `#tableId` | 简单参数 |
| `#参数名?.属性` | `#request?.memberId` | 带安全导航的属性访问 |
| `#参数名.属性` | `#request.name` | 属性访问 |

## 使用示例

```java
@PostMapping("/{id}/end")

    @RequestBody(required = false) EndTableRequest request,
    @RequestHeader(value = "Authorization", required = false) String authHeader)
```

```java
@LogOperation(

)
```
