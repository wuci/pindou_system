# 修复桌台分类时间字段类型问题

## 日期
2026-04-04

## 问题描述
桌台分类管理新增分类时报错：
```
Cause: java.sql.SQLException: Data truncated for column 'created_at' at row 1
```

## 问题原因
`biz_table_category` 和 `biz_table_layout_config` 表的 `created_at` 和 `updated_at` 字段类型在数据库中定义为 `BIGINT`（毫秒时间戳），但 Java 实体类使用的是 `LocalDateTime` 类型，导致类型不匹配。

**类型对比：**
| 表 | 数据库字段类型 | Java 实体类型 | 是否匹配 |
|---|---|---|---|
| biz_table_category | BIGINT | LocalDateTime | ❌ 不匹配 |
| biz_table_layout_config | BIGINT | LocalDateTime | ❌ 不匹配 |

**原始定义：**
在 `add_table_category.sql` 和 `add_table_layout_config.sql` 中，时间字段类型是 `DATETIME`，与实体类匹配。但在 `01_建表脚本.sql` 中被错误地改成了 `BIGINT`。

## 修改内容

### 文件：backend/sql/20260403/01_建表脚本.sql

**biz_table_category 表修改：**
```sql
-- 修改前
`created_at` BIGINT NOT NULL COMMENT '创建时间（毫秒时间戳）',
`updated_at` BIGINT NOT NULL COMMENT '更新时间（毫秒时间戳）',

-- 修改后
`created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
`updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
```

**biz_table_layout_config 表修改：**
```sql
-- 修改前
`created_at` BIGINT NOT NULL COMMENT '创建时间（毫秒时间戳）',
`updated_at` BIGINT NOT NULL COMMENT '更新时间（毫秒时间戳）',

-- 修改后
`created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
`updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
```

### 新建文件：backend/sql/20260404/fix_table_category_time_columns.sql

为已存在的数据库提供迁移脚本，将 `BIGINT` 类型的时间字段转换为 `DATETIME` 类型。

## 实体类说明

### TableCategory.java
```java
@TableField("created_at")
private LocalDateTime createdAt;

@TableField("updated_at")
private LocalDateTime updatedAt;
```

### TableLayoutConfig.java
```java
@TableField(value = "created_at", fill = FieldFill.INSERT)
private LocalDateTime createdAt;

@TableField(value = "updated_at", fill = FieldFill.INSERT_UPDATE)
private LocalDateTime updatedAt;
```

## 时间字段类型选择说明

| 类型 | 使用场景 | 示例表 |
|------|----------|--------|
| BIGINT | 需要跨时区或精确到毫秒 | sys_user, sys_role, biz_member, biz_order, biz_table |
| DATETIME | 应用层自动填充，使用 LocalDateTime | biz_table_category, biz_table_layout_config |

**选择原则：**
- 使用 MyBatis-Plus 的 `FieldFill.INSERT` 和 `FieldFill.INSERT_UPDATE` 自动填充时，应使用 `DATETIME` 类型
- 手动设置毫秒时间戳时，应使用 `BIGINT` 类型

## 影响范围
- 建表脚本 `backend/sql/20260403/01_建表脚本.sql`
- 需要执行迁移脚本修复现有数据库
