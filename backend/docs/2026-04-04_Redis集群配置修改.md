# Redis 集群配置修改

## 日期
2026-04-04

## 修改内容

### 文件：backend/pdsystem-server/src/main/resources/application.yml

将 Redis 配置从单机模式改为集群模式。

**修改前（单机模式）：**
```yaml
spring:
  redis:
    host: ${REDIS_HOST:localhost}
    port: ${REDIS_PORT:6379}
    password: ${REDIS_PASSWORD:}
    database: 0
```

**修改后（集群模式）：**
```yaml
spring:
  redis:
    cluster:
      nodes:
        - 8.130.49.28:7000
        - 8.130.49.28:7001
        - 8.130.49.28:7002
        - 8.130.49.28:7003
        - 8.130.49.28:7004
        - 8.130.49.28:7005
      max-redirects: 3
    password: ${REDIS_PASSWORD:}
```

## 配置说明

| 配置项 | 说明 |
|--------|------|
| nodes | Redis 集群节点列表（host:port 格式） |
| max-redirects | 最大重定向次数，当集群发生槽迁移时使用 |
| password | Redis 集群密码（如有） |

## 注意事项

1. **集群模式不支持多数据库**：Redis 集群模式不支持 `database` 配置，只能使用 db0

2. **连接池配置**：Lettuce 连接池配置保持不变，同样适用于集群模式

3. **环境变量**：可通过环境变量 `REDIS_PASSWORD` 设置密码

4. **节点配置**：确保所有节点都是可访问的，否则可能导致连接失败

## 项目信息
- Spring Boot 版本：2.7.18
- Spring Data Redis 版本：由 spring-boot-starter-parent 管理

## 测试建议
1. 启动应用，观察 Redis 连接日志
2. 调用需要 Redis 的接口（如登录、缓存等）
3. 检查是否有连接错误或超时

## 故障排查

如果遇到连接问题：
1. 检查 Redis 集群是否正常运行
2. 确认网络能访问所有节点
3. 检查密码是否正确
4. 查看 `RedisConfig.java` 配置是否正确
