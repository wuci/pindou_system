# 前后端一体化打包实现

## 日期
2026-04-04

## 功能说明
将前端 Vue 应用打包集成到后端 Spring Boot 应用中，实现一个 jar 包同时提供前端页面和后端 API 服务。

## 修改文件清单

### 1. 后端配置类
**文件：** `backend/pdsystem-server/src/main/java/com/pindou/timer/config/WebMvcConfig.java`

**修改内容：**
- 添加 `addResourceHandlers()` 方法，配置静态资源映射
- 将 `classpath:/static/` 映射到根路径 `/**`
- 在拦截器排除路径中添加前端相关路径

### 2. SPA 路由回退控制器
**文件：** `backend/pdsystem-server/src/main/java/com/pindou/timer/controller/ETSSpaFallbackController.java`

**功能说明：**
- 处理 Vue Router 的 history 模式路由
- 当访问的前端路由（如 `/dashboard`、`/tables` 等）时返回 `index.html`
- 支持根路径 `/` 和登录页 `/login` 的访问

### 3. 前端构建配置
**文件：** `frontend/vite.config.ts`

**修改内容：**
- 生产环境构建输出到 `../backend/pdsystem-server/src/main/resources/static`
- 开发环境保持输出到 `dist`

### 4. Maven 构建配置
**文件：** `backend/pdsystem-server/pom.xml`

**修改内容：**
- 添加 `frontend-maven-plugin` 插件
- 配置在 `generate-resources` 阶段自动构建前端
- 使用 npm 镜像源加速依赖安装

## 构建方式

### 方式一：使用 Maven 构建（推荐）
```bash
# 在 backend 目录下执行
cd backend
mvn clean package -DskipTests
```

构建完成后，jar 包位置：
```
backend/pdsystem-server/target/pdsystem-server-1.0.0.jar
```

### 方式二：分别构建
```bash
# 1. 构建前端
cd frontend
npm run build

# 2. 构建后端
cd ../backend
mvn clean package -DskipTests
```

## 启动方式

```bash
# 直接运行 jar 包
java -jar backend/pdsystem-server/target/pdsystem-server-1.0.0.jar

# 或指定配置文件
java -jar backend/pdsystem-server/target/pdsystem-server-1.0.0.jar --spring.config.location=config/application.yml
```

启动后访问：
- 前端页面：`http://localhost:9026/`
- 接口文档：`http://localhost:9026/doc.html`
- API 接口：`http://localhost:9026/api/...`

## 构建流程说明

```
Maven 构建流程
    ↓
[generate-resources 阶段]
    ↓
安装 Node.js 和 npm
    ↓
执行 npm install（安装前端依赖）
    ↓
执行 npm run build（构建前端）
    ↓
前端资源输出到 resources/static/
    ↓
[compile 阶段]
    ↓
编译后端代码
    ↓
[package 阶段]
    ↓
打包成可执行 jar 包
```

## 注意事项

1. **首次构建时间较长**：需要下载 Node.js 和安装 npm 依赖

2. **内存要求**：建议至少 4GB 可用内存

3. **网络要求**：
   - 需要访问 Node.js 下载源
   - 需要访问 npm 镜像源（已配置为 npmmirror.com）

4. **开发模式**：
   - 前端开发时仍使用 `npm run dev`（端口 3000）
   - 后端开发时单独启动 Spring Boot
   - 一体化打包仅用于生产环境部署

5. **静态资源缓存**：静态资源缓存时间为 1 小时

## 故障排查

### 问题：前端资源未加载
- 检查 `resources/static/` 目录是否存在
- 检查 `index.html` 是否存在
- 查看启动日志是否有静态资源加载相关错误

### 问题：前端路由 404
- 确认 `ETSSpaFallbackController` 已正确注册
- 检查拦截器是否正确排除了前端路径

### 问题：Maven 构建失败
- 检查 Node.js 版本是否正确下载
- 检查 npm 依赖安装是否成功
- 尝试手动进入 frontend 目录执行 `npm run build`
