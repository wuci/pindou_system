# 2026-04-03 用户管理角色列表加载修复

## 修改日期
2026-04-03

## 问题描述
用户管理页面中，角色下拉框无法加载角色列表。

## 问题原因
1. 前端代码调用了 `getRoleList()` 方法
2. 该方法需要传入分页参数（page、pageSize、pageLimit）
3. 用户管理页面的角色下拉框需要获取所有角色，不需要分页
4. 应该调用 `getAllRoles()` 方法，该方法返回所有启用状态的角色列表

## 修改的文件

### `frontend/src/views/UserManagement/index.vue`

**修改点 1：导入语句（第 185 行）**

修改前：
```typescript
import { getRoleList } from '@/api/role'
```

修改后：
```typescript
import { getAllRoles } from '@/api/role'
```

**修改点 2：函数调用（第 260 行）**

修改前：
```typescript
const loadRoles = async () => {
  try {
    const result = await getRoleList()
    roleList.value = result
  } catch (error) {
    console.error('加载角色列表失败', error)
  }
}
```

修改后：
```typescript
const loadRoles = async () => {
  try {
    const result = await getAllRoles()
    roleList.value = result
  } catch (error) {
    console.error('加载角色列表失败', error)
  }
}
```

## 后端接口说明

### `RoleController.getAllRoles()`

**请求路径**：`GET /roles/all`

**返回数据**：`List<RoleInfoResponse>`

**功能说明**：
- 返回所有启用状态（status=1）的角色
- 不需要分页参数
- 适合用于下拉框选择

**RoleInfoResponse 字段**：
- `id`: 角色ID
- `name`: 角色名称
- `description`: 角色描述
- `status`: 状态（1-启用，0-禁用）

## 测试要点

1. **页面加载**：验证角色下拉框能正常加载角色列表
2. **角色筛选**：验证筛选条件中的角色下拉框正常工作
3. **新增用户**：验证新增用户时能选择角色
4. **编辑用户**：验证编辑用户时能选择角色
5. **仅显示启用角色**：验证只显示启用状态的角色

## 优势

1. ✅ **修复加载失败**：角色列表正常加载
2. ✅ **符合业务逻辑**：获取所有角色用于下拉框选择
3. ✅ **无分页限制**：一次性获取所有角色，避免分页问题
4. ✅ **过滤禁用角色**：只显示启用状态的角色，避免选择已禁用角色
