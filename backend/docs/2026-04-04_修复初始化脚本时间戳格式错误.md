# 修复初始化脚本时间戳格式错误

## 日期
2026-04-04

## 问题描述

执行初始化数据脚本时，插入 `biz_table_category` 表报错：

```
SQL 错误 [1292] [22001]: Data truncation: Incorrect datetime value: '1774596555000' for column 'created_at' at row 1
```

## 原因分析

`biz_table_category` 表的 `created_at` 和 `updated_at` 字段定义为 `DATETIME` 类型：

```sql
CREATE TABLE `biz_table_category` (
  ...
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  ...
);
```

但初始化脚本中插入的是毫秒时间戳数值 `1774596555000`，导致类型不匹配。

## 解决方案

使用 `FROM_UNIXTIME()` 函数将毫秒时间戳转换为 DATETIME 格式：

**修改前：**
```sql
INSERT INTO `biz_table_category` (id, name, icon, sort_order, remark, created_at, updated_at) VALUES
(1, '大厅', 'grid', 1, '大厅区域桌台', 1774596555000, 1774596555000),
(2, '包间', 'office-building', 2, '包间桌台', 1774596555000, 1774596555000);
```

**修改后：**
```sql
INSERT INTO `biz_table_category` (id, name, icon, sort_order, remark, created_at, updated_at) VALUES
(1, '大厅', 'grid', 1, '大厅区域桌台', FROM_UNIXTIME(1774596555000 / 1000), FROM_UNIXTIME(1774596555000 / 1000)),
(2, '包间', 'office-building', 2, '包间桌台', FROM_UNIXTIME(1774596555000 / 1000), FROM_UNIXTIME(1774596555000 / 1000));
```

## 注意事项

1. 不同表的时间字段类型不统一：
   - `biz_table_category`: `DATETIME` 类型
   - `biz_member_level`: `BIGINT` 类型（毫秒时间戳）
   - `biz_table`: `BIGINT` 类型（毫秒时间戳）

2. 建议：统一时间字段类型，避免混用
