# 2026-04-03 角色权限保存问题修复

## 修改日期
2026-04-03

## 问题描述

用户通过角色管理界面配置权限后，保存成功但权限没有生效。

## 问题原因

**文件**：`frontend/src/components/PermissionTree.vue`

`el-tree` 组件的 `getCheckedKeys()` 方法默认只返回叶子节点的 key。

在我们的权限树结构中：
- **模块节点**也是有效的权限节点，有自己的 permission_key（如 `dashboard:view`, `table:view` 等）
- **操作权限节点**是模块的子节点（如 `table:delete`, `table:category` 等）

当用户勾选模块节点时，如果只返回叶子节点，模块节点的权限 key 就会丢失，导致权限不完整。

## 修复方案

修改 `handleCheck` 方法，使用 `getCheckedKeys(false)` 返回所有选中的节点（包括父节点）。

**修改前**：
```typescript
const handleCheck = () => {
  if (!treeRef.value) return
  const checkedKeys = treeRef.value.getCheckedKeys()  // 默认只返回叶子节点
  emit('update:modelValue', checkedKeys as string[])
}
```

**修改后**：
```typescript
const handleCheck = () => {
  if (!treeRef.value) return
  // 获取所有选中的节点（包括父节点），不只是叶子节点
  const checkedKeys = treeRef.value.getCheckedKeys(false)
  emit('update:modelValue', checkedKeys as string[])
}
```

## 权限树结构说明

### 工作台模块
```
工作台 (dashboard:view) - 模块节点，本身也是权限
```

### 桌台管理模块
```
桌台管理 (table:view) - 模块节点，本身也是权限
  ├─ 批量删除 (table:delete)
  ├─ 分类管理 (table:category)
  ├─ 桌台配置 (table:config)
  ├─ 开始计时 (table:start)
  ├─ 结束计时 (table:end)
  ├─ 预定 (table:reserve)
  ├─ 结账 (table:bill)
  ├─ 续费 (table:extend)
  └─ 暂停 (table:pause)
```

### 会员管理模块
```
会员管理 (member:view) - 模块节点，本身也是权限
  ├─ 新增会员 (member:create)
  ├─ 编辑 (member:update)
  ├─ 删除 (member:delete)
  ├─ 充值 (member:recharge)
  └─ 查看记录 (member:record)
```

## 验证方法

### 1. 浏览器控制台调试

保存角色前，在浏览器控制台查看将要保存的权限：

```javascript
// 在角色编辑页面，打开浏览器控制台
// 点击保存前，执行以下代码查看权限列表
console.log('将要保存的权限:', formData.permissions)
```

### 2. 查看保存后的数据库数据

```sql
-- 查看角色权限配置
SELECT
    id,
    name,
    code,
    JSON_PRETTY(permissions) as permissions,
    status
FROM sys_role
WHERE code = 'role_manager';
```

应该看到完整的权限列表，包括模块权限和操作权限。

### 3. 查看当前登录用户的权限

在浏览器控制台执行：

```javascript
import { useUserStore } from '@/stores/user'
const userStore = useUserStore()
console.log('用户权限列表:', userStore.permissions)
console.log('是否有 table:view:', userStore.hasPermission('table:view'))
console.log('是否有 member:view:', userStore.hasPermission('member:view'))
```

## 使用说明

### 保存权限后必须重新登录

**重要**：修改角色权限后，该角色的用户需要：
1. **退出登录**
2. **重新登录**

因为用户的权限信息在登录时加载，保存在 localStorage 中。

### 权限勾选建议

#### 建议一：勾选模块节点（推荐）

勾选模块节点会自动选中该模块下的所有操作权限：

- 勾选 `桌台管理 (table:view)` 会自动选中其下所有操作权限
- 这样配置简单，适合给予某个模块的完整权限

#### 建议二：精确勾选操作权限

如果只需要部分操作权限：
- 不要勾选模块节点
- 只勾选需要的操作权限节点
- 例如：只勾选 `table:view` 和 `table:start`，只给查看和开始计时的权限
