## 修改日期

## 修改内容

```typescript
/**
 * 获取支付方式标签
 */
const getPaymentMethodLabel = (method: string) => {
  const labels: Record<string, string> = {
    offline: '线下支付',
    online: '线上支付',
    balance: '会员余额',
    combined: '组合支付'
  }
  return labels[method] || method
}

/**
 * 获取支付方式标签类型
 */
const getPaymentMethodTagType = (method: string) => {
  const types: Record<string, string> = {
    offline: '',
    online: 'primary',
    balance: 'success',
    combined: 'warning'
  }
  return types[method] || ''
}
```

## 功能说明

### 列宽设置
- 支付方式列：160px
- 宽度足够显示支付方式标签和支付明细

### 显示逻辑

1. **整个支付方式单元格**：
   - 如果 `row.paymentMethod` 存在，显示支付方式信息
   - 否则显示 "-"（灰色）

2. **支付方式标签**：
   - 使用 el-tag 显示支付方式
   - 不同支付方式使用不同颜色标签

3. **支付明细**：
   - 仅当支付方式是 'combined' 或 'balance' 时显示
   - 余额支付金额 > 0 时显示（绿色）
   - 线下支付金额 > 0 时显示（红色）

4. **支付时间**：
   - 仅在历史订单中显示
   - 仅当 `row.paidAt` 存在时显示
   - 显示格式：2026-04-03 14:30:00

### 样式特点

1. **垂直布局**：
   - 支付方式标签在上方
   - 支付明细在下方
   - 支付时间在最下方（仅历史订单）

2. **颜色区分**：
   - 余额支付：绿色文字 + 浅绿色背景
   - 线下支付：红色文字 + 浅红色背景

3. **字体大小**：
   - 标签：默认大小
   - 支付明细：12px 小字
   - 支付时间：11px 灰色小字

## 数据流程

1. **订单创建/更新**：
   - TableServiceImpl 在操作订单时设置支付方式字段
   - 设置 `paymentMethod`、`balanceAmount`、`otherPaymentAmount`
   - 结账时设置 `paidAt` 时间戳

2. **查询订单列表**：
   - OrderServiceImpl 查询订单列表
   - convertToInfoResponse() 转换为响应对象
   - 设置支付方式相关字段到响应对象

3. **前端显示**：
   - OrderManagement 页面接收订单列表数据
   - 在表格中显示支付方式列
   - 根据字段值判断显示哪些信息
   - 应用相应的样式和颜色
