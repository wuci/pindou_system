# 2026-04-03 endTimer 方法重构优化

## 修改日期
2026-04-03

## 修改内容
重构 `TableServiceImpl.endTimer()` 方法，将235行的超长方法拆分为多个职责单一的私有方法，提高代码的可读性、可维护性和可测试性。

## 重构前的问题

1. **方法过长**：235行代码，难以理解和维护
2. **职责不清**：一个方法承担了多个职责（验证、计算、更新、扣款等）
3. **可读性差**：代码嵌套层次深，逻辑复杂
4. **可测试性差**：无法单独测试各个逻辑块
5. **可复用性差**：逻辑耦合在一起，无法复用

## 重构后的结构

### 主方法 `endTimer()` (30行)

```java
public void endTimer(Integer tableId, EndTableRequest request, String userId, String username) {
    Long memberId = request != null ? request.getMemberId() : null;
    log.info("结束计时: tableId={}, userId={}, memberId={}", tableId, userId, memberId);

    // 1. 验证并获取桌台
    Table table = validateTableForEndTimer(tableId);
    long now = System.currentTimeMillis();

    // 2. 更新订单状态和计算费用
    if (table.getCurrentOrderId() != null) {
        Order order = orderMapper.selectById(table.getCurrentOrderId());
        if (order != null) {
            // 2.1 计算时长
            DurationCalculation durationCalc = calculateDurations(table, now);

            // 2.2 判断订单状态
            String orderStatus = determineOrderStatus(durationCalc.getActualDuration());

            // 2.3 计算费用并应用折扣
            OrderAmountCalculation amountCalc = calculateOrderAmount(order, table, durationCalc, memberId);

            // 2.4 构建金额明细JSON
            String amountDetailJson = buildAmountDetailJson(amountCalc.getAmountDetail(), durationCalc.getActualDuration());

            // 2.5 更新订单
            updateOrderForEndTimer(order, durationCalc, amountCalc, amountDetailJson, orderStatus, request, now, memberId);

            // 2.6 处理余额扣除
            processBalanceDeduction(order, amountCalc.getFinalAmount(), orderStatus);

            // 2.7 保存订单
            orderMapper.updateById(order);

            // 2.8 更新会员累计消费
            updateMemberTotalAmountIfNeeded(memberId, amountCalc.getOriginalAmount(), orderStatus);

            log.info("订单已结束: ...");
        }
    }

    // 3. 重置桌台状态
    resetTableAfterEndTimer(table, now);

    // 4. 推送桌台状态变更
    eventPublisher.publishEvent(new TableStatusChangeEvent(this, tableId));

    log.info("结束计时成功: tableId={}", tableId);
}
```

### 拆分的私有方法

#### 1. `validateTableForEndTimer(Integer)` - 验证桌台状态
- **职责**：验证桌台是否存在且状态正确
- **返回**：桌台信息
- **行数**：约15行

#### 2. `calculateDurations(Table, long)` - 计算时长
- **职责**：计算总时长、暂停时长、实际时长
- **返回**：`DurationCalculation` 对象
- **行数**：约20行

#### 3. `determineOrderStatus(int)` - 判断订单状态
- **职责**：根据实际时长判断订单是完成还是作废
- **返回**：订单状态
- **行数**：约15行

#### 4. `calculateOrderAmount(...)` - 计算订单费用
- **职责**：计算费用并应用会员折扣
- **返回**：`OrderAmountCalculation` 对象
- **行数**：约50行

#### 5. `applyMemberDiscountIfNeeded(...)` - 应用会员折扣
- **职责**：判断是否需要应用折扣并执行
- **参数**：订单信息、会员ID、费用计算结果
- **行数**：约30行

#### 6. `buildAmountDetailJson(...)` - 构建金额明细JSON
- **职责**：将费用明细转换为JSON字符串
- **返回**：JSON字符串
- **行数**：约15行

#### 7. `updateOrderForEndTimer(...)` - 更新订单
- **职责**：设置订单的所有字段
- **参数**：9个参数（订单、时长、费用、JSON、状态、请求、时间、会员ID）
- **行数**：约20行

#### 8. `processBalanceDeduction(...)` - 处理余额扣除
- **职责**：处理会员余额扣除和消费记录
- **参数**：订单信息、最终金额、订单状态
- **行数**：约60行

#### 9. `updateMemberTotalAmountIfNeeded(...)` - 更新会员累计消费
- **职责**：根据订单状态更新会员累计消费
- **参数**：会员ID、原价、订单状态
- **行数**：约10行

#### 10. `resetTableAfterEndTimer(...)` - 重置桌台状态
- **职责**：重置桌台所有字段为初始状态
- **参数**：桌台信息、当前时间戳
- **行数**：约15行

## 重构优势

### 1. 可读性提升 ✅
- **主方法只有30行**，一目了然地看到整个流程
- **方法命名清晰**，每个方法名都说明了它的职责
- **逻辑分层明确**，从验证到计算到更新，层层递进

### 2. 可维护性提升 ✅
- **单一职责**，每个方法只做一件事
- **修改局部化**，修改某个逻辑只需修改对应方法
- **减少嵌套**，代码层次更清晰

### 3. 可测试性提升 ✅
- **可单独测试**，每个私有方法都可以独立测试
- **依赖注入**，通过参数传递依赖，便于mock
- **返回结果**，通过返回值验证结果

### 4. 可复用性提升 ✅
- **方法提取**，这些方法可以在其他地方复用
- **逻辑独立**，减少代码重复

### 5. 符合设计原则 ✅
- **单一职责原则（SRP）**：每个方法只负责一个功能
- **开闭原则（OCP）**：对扩展开放，对修改关闭
- **依赖倒置原则（DIP）**：依赖抽象而非具体实现
