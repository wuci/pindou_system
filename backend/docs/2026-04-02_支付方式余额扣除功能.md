## 功能说明

### 支付流程
1. **开始计时**：用户选择支付方式（线下/线上/会员余额/组合支付）
2. **续费**：用户可以选择或修改支付方式
3. **结账**：系统显示账单，用户确认结账时，前端将支付方式传递给后端
4. **余额扣除**：后端根据支付方式自动扣除会员余额（如果是余额或组合支付）

### 支付方式说明
- **线下支付 (offline)**：店内现场支付，不扣除余额
- **线上支付 (online)**：平台订单支付，不扣除余额
- **会员余额 (balance)**：使用会员余额支付，余额不足时无法选择
- **组合支付 (combined)**：余额+线下组合支付，余额不足时自动计算

## 测试要点
1. 非会员结账：选择线下/线上支付，确认结账成功
2. 会员结账 - 线下支付：选择线下支付，确认余额未扣除
3. 会员结账 - 余额支付：选择会员余额，确认余额正确扣除
4. 会员结账 - 组合支付：选择组合支付，确认余额部分扣除，剩余金额线下支付
5. 余额不足：会员余额不足时，组合支付显示正确的拆分金额
