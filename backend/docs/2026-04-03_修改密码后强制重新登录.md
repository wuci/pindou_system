# 修改密码后强制重新登录

**日期：** 2026-04-03
**修改人：** wuci
**优化说明：** 修改密码成功后，如果是修改当前登录用户的密码，需要退出登录并跳转到登录页面

## 安全考虑

修改密码是敏感操作，特别是修改当前登录用户的密码时，出于安全考虑，需要用户重新登录：

1. **防止会话劫持**：旧密码失效后，原会话应该立即失效
2. **确认新密码**：强制用户使用新密码重新登录，确保密码修改成功
3. **防止误操作**：给用户明确的提示，确认密码已被修改

## 修改内容

### 文件：`frontend/src/views/UserManagement/index.vue`

#### 1. 导入 useRouter


#### 2. 添加 router 实例

```typescript
const userStore = useUserStore()
const router = useRouter()
```

#### 3. 修改提交函数逻辑

```typescript
// 提交修改密码
const handleChangePasswordSubmit = async () => {
  if (!passwordFormRef.value) return

  try {
    await passwordFormRef.value.validate()
    passwordSubmitting.value = true

    await changePassword({
      userId: passwordFormData.userId,
      newPassword: passwordFormData.newPassword
    })

    ElMessage.success('修改密码成功')

    // 如果修改的是当前登录用户的密码，需要退出登录
    if (passwordFormData.userId === userStore.userInfo?.id) {
      await ElMessageBox.confirm(
        '密码已修改，需要重新登录',
        '提示',
        {
          confirmButtonText: '确定',
          cancelButtonText: '取消',
          type: 'info',
          showCancelButton: false,
          closeOnClickModal: false,
          closeOnPressEscape: false
        }
      )

      // 退出登录并跳转到登录页
      await userStore.logout()
      router.push('/login')
    } else {
      passwordDialogVisible.value = false
    }
  } catch (error) {
    if (error !== false) {
      ElMessage.error('修改密码失败')
    }
  } finally {
    passwordSubmitting.value = false
  }
}
```

## 实现逻辑

### 判断逻辑

```typescript
if (passwordFormData.userId === userStore.userInfo?.id) {
```

### 提示框配置

- **不可关闭**：`closeOnClickModal: false, closeOnPressEscape: false`
- **无取消按钮**：`showCancelButton: false`
- **强制确认**：用户必须点击"确定"才能继续

### 退出登录流程

1. 调用 `userStore.logout()` 清除本地登录状态
2. 调用 `router.push('/login')` 跳转到登录页

## 安全特性

1. **即时生效**：密码修改成功后立即退出登录
2. **不可绕过**：提示框不可通过其他方式关闭
3. **明确提示**：用户清楚知道需要重新登录
4. **状态清理**：完全清除本地登录状态和 token
