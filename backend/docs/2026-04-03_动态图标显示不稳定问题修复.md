# 2026-04-03 动态图标显示不稳定问题修复

## 修改日期
2026-04-03

## 问题描述
系统中的分类图标（如桌台分类图标）显示不稳定，有时显示有时不显示。

## 问题原因

### 问题 1：图标组件未导入

Vue 3 的动态组件 `<component :is="xxx" />` 需要组件已经被导入和注册才能正确渲染。多个文件中使用动态组件显示图标，但没有导入相应的图标组件，导致：

1. Vue 无法找到对应的组件定义
2. 组件渲染失败或渲染异常
3. 图标显示不稳定

### 问题 2：使用了不存在的图标名称

Element Plus icons-vue 中部分图标名称不存在：
- `Dashboard` ❌ - 不存在，应使用 `Odometer` 或 `Monitor` 或 `DataBoard`
- `Tickets` ❌ - 不存在，应使用 `Notebook` 或 `DocumentCopy`

使用了无效的图标名称会导致运行时错误：
```
SyntaxError: The requested module does not provide an export named 'Dashboard'
```

### 问题 3：Setting 图标组件本身有渲染问题 ⭐ **最终问题**

**测试结果**：
- 使用 Star 图标替换 Setting 后，图标不再闪烁
- 使用 Tools 图标替换 Setting 后，图标不再闪烁
- 说明问题出在 Setting 图标组件本身

**根本原因**：
Element Plus 的 `Setting` 图标组件可能存在：
1. 内部渲染逻辑问题
2. CSS 动画或过渡效果
3. SVG 路径渲染问题
4. 组件初始化时的状态切换

**最终修复方案**：
```typescript
// ❌ 有问题的图标
import { Setting } from '@element-plus/icons-vue'
<el-icon><Setting /></el-icon>  <!-- 闪烁 -->

// ✅ 替换为稳定的图标
import { Tools } from '@element-plus/icons-vue'
<el-icon><Tools /></el-icon>  <!-- 稳定 -->
```

**修改内容**：
1. 将 `Setting` 图标替换为 `Tools` 图标
2. 保留 computed 缓存的权限判断（优化性能）
3. 移除调试代码

**Element Plus 图标问题**：
某些 Element Plus 图标可能存在渲染问题，建议：
- Setting → 推荐使用 Tools 或 Operation 代替
- 如果遇到图标闪烁，尝试替换为其他相似图标

## 修复方案

### 核心思路
1. **导入图标组件**：在 script setup 中导入所有可能使用的图标组件
2. **创建图标映射**：使用对象映射图标名称到组件实例
3. **使用映射渲染**：修改模板中的动态组件，使用映射对象而不是字符串

### 修复步骤

#### 步骤 1：导入图标组件

```typescript
// 修复前
import { Plus, Edit, Delete } from '@element-plus/icons-vue'

// 修复后
import { Plus, Edit, Delete, Grid, List, Star, OfficeBuilding, House } from '@element-plus/icons-vue'
```

#### 步骤 2：创建图标映射对象

```typescript
// 在组件中添加图标映射
const iconMap = {
  Grid,
  List,
  Star,
  OfficeBuilding,
  House
}
```

#### 步骤 3：修改模板渲染逻辑

```vue
<!-- 修复前 -->
<el-icon v-if="category.icon" class="category-icon">
  <component :is="category.icon" />
</el-icon>

<!-- 修复后 -->
<el-icon v-if="category.icon && iconMap[category.icon]" class="category-icon">
  <component :is="iconMap[category.icon]" />
</el-icon>
```

## 技术原理

### Vue 3 动态组件

Vue 3 的 `<component :is="xxx">` 有两种使用方式：

**方式 1：直接使用组件实例**
```vue
<component :is="MyComponent" />
```
- 要求 `MyComponent` 已经导入
- 适用于已知的固定组件

**方式 2：使用字符串名称**
```vue
<component :is="'MyComponent'" />
```
- 要求组件已经在全局注册
- 或者在组件选项中通过 `components` 选项注册

**方式 3：使用对象映射（推荐）**
```vue
<component :is="componentMap['MyComponent']" />
```
- 通过映射对象获取组件实例
- 确保组件已被导入
- 类型安全，可维护性好

### 为什么需要图标映射

1. **确保组件已导入**：在 `<script setup>` 中导入的组件会自动注册
2. **类型安全**：TypeScript 可以检查映射对象中的键值
3. **避免全局污染**：不需要全局注册组件
4. **便于维护**：集中管理所有图标映射

## 优势

1. ✅ **修复显示问题**：图标稳定显示，不再时有时无
2. ✅ **类型安全**：通过映射对象确保图标名称正确
3. ✅ **性能优化**：Vue 可以正确识别和优化组件渲染
4. ✅ **易于扩展**：新增图标只需添加到映射对象中
5. ✅ **防御性编程**：添加 `iconMap[category.icon]` 检查，避免渲染不存在的图标

## 测试要点

1. **分类图标显示**：
   - 新增分类时选择不同图标，验证图标正确显示
   - 编辑分类时修改图标，验证图标更新
   - 刷新页面后验证图标依然显示

2. **分类标签图标**：
   - 切换不同分类，验证分类标签中的图标正确显示
   - 刷新页面后验证图标依然显示

3. **边界情况**：
   - 没有设置图标的分类，验证不显示图标区域
   - 设置了无效图标名称，验证不报错且不显示图标
