# 2026-04-03 无效订单时间配置优化

## 修改日期
2026-04-03

## 修改内容
将结账时的无效订单时间判断从硬编码的 300 秒（5分钟）改为从系统参数配置中读取，支持动态配置。

## 问题描述
原代码在 `TableServiceImpl.endTimer()` 方法中，硬编码了 300 秒作为无效订单的判断阈值：

```java
if (actualDuration < 300) { // 5分钟 = 300秒
    orderStatus = "cancelled";
}
```

这种硬编码方式存在以下问题：
1. 不够灵活，无法根据业务需求调整
2. 修改时间需要重新编译部署
3. 与系统配置脱节，其他模块已支持配置化

## 修改的文件

### 后端文件

#### 1. `backend/pdsystem-service/src/main/java/com/pindou/timer/service/impl/TableServiceImpl.java`

**修改点 1：优化订单状态判断逻辑（第 561-572 行）**

修改前：
```java
// 判断订单状态：使用时长小于5分钟的订单标记为作废
String orderStatus;
if (actualDuration < 300) { // 5分钟 = 300秒
    orderStatus = "cancelled";
    log.info("订单使用时长不足5分钟，标记为作废: orderId={}, actualDuration={}", order.getId(), actualDuration);
} else {
    orderStatus = "completed";
}
```

修改后：
```java
// 判断订单状态：从系统配置获取无效订单时间（分钟），转换为秒进行判断
int invalidOrderThresholdSeconds = getInvalidOrderThresholdInSeconds();
String orderStatus;
if (invalidOrderThresholdSeconds > 0 && actualDuration < invalidOrderThresholdSeconds) {
    orderStatus = "cancelled";
    log.info("订单使用时长不足{}分钟，标记为作废: orderId={}, actualDuration={}秒",
            invalidOrderThresholdSeconds / 60, order.getId(), actualDuration);
} else {
    orderStatus = "completed";
}
```

**修改点 2：添加辅助方法获取无效订单时间阈值（第 908-936 行）**

```java
/**
 * 获取无效订单时间阈值（秒）
 * 从系统配置中获取无效订单时间（分钟），转换为秒返回
 * 如果配置为0或不存在，返回0表示不限制
 *
 * @return 无效订单时间阈值（秒）
 */
private int getInvalidOrderThresholdInSeconds() {
    try {
        String systemConfigJson = configService.getSystemConfig();
        if (systemConfigJson != null && !systemConfigJson.isEmpty()) {
            cn.hutool.json.JSONObject jsonObject = cn.hutool.json.JSONUtil.parseObj(systemConfigJson);
            Integer invalidOrderTimeMinutes = jsonObject.getInt("invalidOrderTime");
            if (invalidOrderTimeMinutes != null && invalidOrderTimeMinutes > 0) {
                int thresholdSeconds = invalidOrderTimeMinutes * 60;
                log.info("从系统配置获取无效订单时间阈值: {}分钟 = {}秒", invalidOrderTimeMinutes, thresholdSeconds);
                return thresholdSeconds;
            }
        }
    } catch (Exception e) {
        log.warn("解析系统配置失败，使用默认值0（不限制）: {}", e.getMessage());
    }
    return 0; // 默认不限制
}
```

## 功能说明

### 系统参数配置
系统参数配置（`system_config`）中包含 `invalidOrderTime` 字段：
- **字段名称**：`invalidOrderTime`
- **单位**：分钟
- **范围**：0-60 分钟
- **默认值**：0（表示不限制，所有订单都正常完成）

### 业务逻辑
1. **配置为 0**：不限制，所有订单都正常完成（状态为 `completed`）
2. **配置为正数**：使用时长小于配置时间的订单标记为作废（状态为 `cancelled`）
3. **配置不存在或解析失败**：默认为 0，不限制

### 与余额扣除的关系
- 订单状态为 `completed` 时，才进行余额扣除
- 订单状态为 `cancelled` 时，不扣除余额

### 配置示例
```json
{
  "extendTime": 30,
  "invalidOrderTime": 3
}
```
表示：延长时间 30 分钟，无效订单时间 3 分钟（使用时长小于 3 分钟的订单标记为作废）

## 优势
1. ✅ **配置灵活**：可通过系统配置界面动态调整，无需重新部署
2. ✅ **代码规范**：消除了魔法值，使用配置化的方式
3. ✅ **向后兼容**：默认值为 0（不限制），不影响现有业务
4. ✅ **异常处理**：配置解析失败时使用默认值，保证系统稳定性
5. ✅ **日志完善**：记录配置读取和判断过程，便于排查问题

## 测试要点
1. **配置为 0**：验证所有订单都正常完成
2. **配置为 3 分钟**：验证小于 3 分钟的订单标记为作废
3. **配置解析失败**：验证使用默认值 0，不影响业务
4. **余额扣除**：验证只有 `completed` 状态的订单才扣除余额
