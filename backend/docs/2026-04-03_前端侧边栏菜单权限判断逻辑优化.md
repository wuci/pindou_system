# 前端侧边栏菜单权限判断逻辑优化

**日期：** 2026-04-03
**修改人：** wuci
**问题描述：** 店长用户配置了具体操作权限（如 `table:start`、`member:create`），但前端侧边栏不显示对应菜单

## 问题分析

### 原有权限判断逻辑（存在问题）

```javascript
const permissions = computed(() => ({
  canViewTables: userStore.hasPermission('table:view'),      // 必须有 table:view
  canViewMembers: userStore.hasPermission('member:view'),    // 必须有 member:view
  canViewSettings: userStore.hasPermission('system:view')    // 必须有 system:view
}))
```

### 用户实际权限

```json
[
    "table:category",     // 桌台分类
    "table:config",       // 桌台配置
    "table:start",        // 开始计时
    "table:end",          // 结束计时
    "table:reserve",      // 预约
    "table:bill",         // 结账
    "table:extend",       // 延长
    "table:pause",        // 暂停
    "member:create",      // 创建会员
    "member:update",      // 更新会员
    "member:recharge",    // 会员充值
    "member:record",      // 会员记录
    "system:rule"         // 系统规则
]
```

### 问题结果

| 菜单 | 用户权限 | 前端要求 | 结果 |
|-----|---------|---------|------|
| 桌台管理 | `table:category` 等8个权限 | 必须有 `table:view` | ❌ 不显示 |
| 会员管理 | `member:create` 等4个权限 | 必须有 `member:view` | ❌ 不显示 |
| 系统设置 | `system:rule` | 必须有 `system:view` | ❌ 不显示 |

**问题本质：** 前端强制要求必须有 `:view` 后缀的权限才显示菜单，这是不合理的。用户配置了模块内的具体操作权限，理应能看到对应的菜单入口。

## 解决方案

### 优化后的权限判断逻辑

```javascript
/**
 * 检查是否有指定模块下的任意权限
 * @param prefix 权限前缀，如 'table:'、'member:'
 */
const hasModulePermission = (prefix: string): boolean => {
  if (!userStore.permissions || userStore.permissions.length === 0) {
    return false
  }
  // 超级管理员
  if (userStore.permissions.includes('*')) {
    return true
  }
  // 检查是否有任意以该前缀开头的权限
  return userStore.permissions.some(p => p.startsWith(prefix))
}

const permissions = computed(() => ({
  canViewDashboard: userStore.hasPermission('dashboard:view'),
  // 桌台管理：有任意 table: 开头的权限即可显示
  canViewTables: hasModulePermission('table:'),
  canViewOrders: userStore.hasPermission('order:view'),
  canViewStatistics: userStore.hasPermission('statistics:view'),
  canViewUsers: userStore.hasPermission('user:view'),
  canViewRoles: userStore.hasPermission('role:view'),
  canViewLogs: userStore.hasPermission('log:view'),
  // 会员管理：有任意 member: 开头的权限即可显示
  canViewMembers: hasModulePermission('member:'),
  // 会员等级：有任意 member:level: 开头的权限即可显示
  canViewMemberLevels: hasModulePermission('member:level:'),
  // 系统设置：有任意 system: 开头的权限即可显示
  canViewSettings: hasModulePermission('system:')
}))
```

### 优化后效果

| 菜单 | 用户权限 | 判断逻辑 | 结果 |
|-----|---------|---------|------|
| 桌台管理 | `table:category` 等 | 有 `table:` 开头的权限 | ✅ 显示 |
| 会员管理 | `member:create` 等 | 有 `member:` 开头的权限 | ✅ 显示 |
| 系统设置 | `system:rule` | 有 `system:` 开头的权限 | ✅ 显示 |

## 影响范围

- 影响所有非超级管理员用户的侧边栏菜单显示
- 只要用户拥有某个模块下的任意权限，即可看到对应菜单
- 具体按钮级别的权限控制在各自的页面组件中继续生效
