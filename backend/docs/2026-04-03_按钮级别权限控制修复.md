# 2026-04-03 按钮级别权限控制修复

## 修改日期
2026-04-03

## 问题描述

系统中的管理页面没有进行按钮级别的权限控制，导致即使用户角色没有相应权限，也能看到并点击操作按钮。

### 问题现象

- 店长账号没有配置 `member:delete` 权限
- 但在会员管理页面仍能看到"删除"按钮
- 点击删除后虽然后端会拦截，但前端体验不好

## 问题原因

前端管理页面只做了**菜单级权限控制**（控制页面访问），没有做**按钮级权限控制**（控制操作按钮显示）。

## 修复方案

在各个管理页面中添加按钮级权限控制：
1. 导入 `useUserStore`
2. 使用 `computed` 定义权限检查
3. 在按钮上添加 `v-if` 指令

## 修改的文件

### 1. 会员管理页面
**文件**：`frontend/src/views/MemberManagement/index.vue`

**新增权限检查**：
```typescript
const userStore = useUserStore()

const permissions = computed(() => ({
  canCreate: userStore.hasPermission('member:create'),
  canUpdate: userStore.hasPermission('member:update'),
  canDelete: userStore.hasPermission('member:delete'),
  canRecharge: userStore.hasPermission('member:recharge')
}))
```

**按钮权限控制**：
```vue
<!-- 新增按钮 -->
<el-button v-if="permissions.canCreate" type="primary" @click="handleAdd">
  新增会员
</el-button>

<!-- 充值按钮 -->
<el-button v-if="permissions.canRecharge" type="success" link @click="handleRecharge(row)">
  充值
</el-button>

<!-- 编辑按钮 -->
<el-button v-if="permissions.canUpdate" type="primary" link @click="handleEdit(row)">
  编辑
</el-button>

<!-- 删除按钮 -->
<el-button v-if="permissions.canDelete" type="danger" link @click="handleDelete(row)">
  删除
</el-button>
```

## 权限检查机制

### 前端权限检查

```typescript
// userStore.hasPermission 实现
hasPermission: (state) => (permission: string | string[]) => {
  // 未登录视为无权限
  if (!state.token || !state.userInfo) {
    return false
  }

  // 超级管理员拥有所有权限
  if (state.permissions.includes('*')) {
    return true
  }

  // 检查是否拥有指定权限
  const perms = Array.isArray(permission) ? permission : [permission]
  return perms.some(p => state.permissions.includes(p))
}
```

### 后端权限校验

前端权限控制只是**UI层面的优化**，真正的权限校验在后端：
- 前端：控制按钮显示，提升用户体验
- 后端：接口级权限校验，保证安全

## 修复效果

| 有删除权限 | ✅ 显示 | 正常操作 ✅ |

### 修复后
| 角色权限 | 前端按钮显示 | 点击后行为 |
|----------|-------------|-----------|
| 无删除权限 | ❌ 不显示 | - |
| 有删除权限 | ✅ 显示 | 正常操作 ✅ |

## 测试要点

1. **权限控制测试**：
   - 登录无删除权限的账号
   - 验证删除按钮不显示
   - 登录有删除权限的账号
   - 验证删除按钮显示且可用

2. **动态权限切换**：
   - 给角色添加/移除权限
   - 刷新页面验证按钮显示/隐藏

3. **多权限组合**：
   - 同时验证多个权限按钮
   - 确保各权限独立控制
