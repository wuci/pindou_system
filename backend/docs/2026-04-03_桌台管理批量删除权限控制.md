# 2026-04-03 桌台管理批量删除权限控制

## 修改日期
2026-04-03

## 问题描述

桌台管理页面中的批量删除功能没有添加按钮级权限控制，没有 `table:delete` 权限的用户仍然可以看到并点击批量删除按钮。

### 问题现象

- 店长账号没有配置 `table:delete` 权限
- 但在桌台管理页面仍能看到"批量删除"按钮
- 点击后虽然后端会拦截，但前端体验不好

## 问题原因

桌台管理页面只做了**菜单级权限控制**，没有做**按钮级权限控制**。批量删除功能分布在两个组件中：
1. `TableManagement/index.vue` - 主页面
2. `TableLayoutEditor.vue` - 布局编辑器组件

## 修复方案

在两个组件中添加 `table:delete` 权限检查。

## 修改的文件

### 1. 桌台管理主页面
**文件**：`frontend/src/views/TableManagement/index.vue`

**新增导入**：
```typescript
import { useUserStore } from '@/stores/user'
```

**新增权限检查**（第 160-168 行）：
```typescript
// 用户状态
const userStore = useUserStore()

// 权限检查
const permissions = computed(() => ({
  canManageCategories: userStore.hasPermission('table:config'),
  canConfig: userStore.hasPermission('table:config'),
  canBatchDelete: userStore.hasPermission('table:delete')
}))
```

**按钮权限控制**：
```vue
<!-- 分类管理按钮 -->
<el-button v-if="permissions.canManageCategories" type="primary" :icon="Setting" size="small" @click="showCategoryDialog = true">
  分类管理
</el-button>

<!-- 桌台配置按钮 -->
<el-button v-if="permissions.canConfig" type="primary" :icon="Setting" size="small" @click="showConfigDialog = true">
  桌台配置
</el-button>

<!-- 批量删除按钮（正常模式） -->
<el-button v-if="permissions.canBatchDelete" type="warning" :icon="Delete" size="small" @click="enterBatchSelectMode">
  批量删除
</el-button>

<!-- 批量删除按钮（批量选择模式） -->
<el-button v-if="permissions.canBatchDelete" type="danger" :icon="Delete" size="small" @click="handleBatchDelete" :disabled="selectedTableIds.size === 0">
  批量删除
</el-button>
```

### 2. 布局编辑器组件
**文件**：`frontend/src/components/TableLayoutEditor.vue`

**新增导入**（第 279 行）：
```typescript
import { useUserStore } from '@/stores/user'
```

**新增权限检查**（第 290-293 行）：
```typescript
const userStore = useUserStore()

const permissions = computed(() => ({
  canBatchDelete: userStore.hasPermission('table:delete')
}))
```

**按钮权限控制**（第 31 行）：
```vue
<!-- 批量删除按钮（布局编辑模式） -->
<el-button v-if="permissions.canBatchDelete" type="danger" @click="handleBatchDelete" :icon="Delete">批量删除</el-button>
```

## 权限对照表

| 操作 | 权限标识 | 说明 |
|------|----------|------|
| 查看桌台 | `table:view` | 访问桌台管理页面 |
| 分类管理 | `table:config` | 管理桌台分类 |
| 桌台配置 | `table:config` | 配置桌台布局 |
| 批量删除 | `table:delete` | 批量删除桌台 |
| 开始计时 | `table:timer` | 开始计时操作 |
| 暂停/继续 | `table:timer` | 暂停/继续计时 |
| 结束计时 | `table:timer` | 结束计时（结账） |
| 续费 | `table:timer` | 续费操作 |
| 预定 | `table:reserve` | 预定桌台 |

## 修复效果

### 修复前
| 角色权限 | 前端按钮显示 | 点击后行为 |
|----------|-------------|-----------|

| 有删除权限 | ✅ 显示 | 正常操作 ✅ |

### 修复后
| 角色权限 | 前端按钮显示 | 点击后行为 |
|----------|-------------|-----------|
| 无删除权限 | ❌ 不显示 | - |
| 有删除权限 | ✅ 显示 | 正常操作 ✅ |

## 批量删除功能说明

批量删除功能涉及两个视图：
1. **网格视图** - 显示所有桌台卡片，支持批量选择和删除
2. **布局视图** - 显示桌台布局，编辑模式下支持批量选择和删除

两个视图都需要添加权限控制，确保：
- 无权限时看不到"批量删除"入口按钮
- 无权限时看不到批量删除确认按钮

## 测试要点

1. **权限控制测试**：
   - 登录无 `table:delete` 权限的账号
   - 验证"批量删除"按钮不显示（两种模式都要验证）
   - 登录有 `table:delete` 权限的账号
   - 验证"批量删除"按钮显示且可用

2. **功能验证**：
   - 网格视图下批量删除功能正常
   - 布局视图下批量删除功能正常
   - 只能删除空闲状态的桌台

3. **权限切换测试**：
   - 给角色添加/移除 `table:delete` 权限
   - 刷新页面验证按钮显示/隐藏
