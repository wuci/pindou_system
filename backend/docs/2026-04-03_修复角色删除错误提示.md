# 修复角色删除错误提示

**日期：** 2026-04-03
**修改人：** wuci
**问题描述：** 角色管理删除时只显示"删除失败"，没有显示具体错误原因

## 问题分析

### 前端错误处理问题

**修改前的代码：**
```typescript
catch (error) {
  if (error !== 'cancel') {
    ElMessage.error('删除失败')  // 只显示固定提示
  }
}
```

**响应拦截器返回的错误结构：**
```typescript
// request.ts 第62行
return Promise.reject(new Error(message || '请求失败'))
```

后端返回的错误信息被包装成了 `Error` 对象，错误信息在 `error.message` 中。

### 可能的删除失败原因

后端有三个删除验证：

1. **角色不存在** (ErrorCode: 11001)
   ```java
   if (role == null) {
     throw new BusinessException(ErrorCode.ROLE_NOT_FOUND, "角色不存在");
   }
   ```

2. **系统内置角色** (ErrorCode: 11003)
   ```java
   if (role.getIsBuiltIn() == 1) {
     throw new BusinessException(ErrorCode.ROLE_SYSTEM_BUILTIN, "系统内置角色不能删除");
   }
   ```

3. **角色正在使用中** (ErrorCode: 11004)
   ```java
   if (userCount > 0) {
     throw new BusinessException(ErrorCode.ROLE_IN_USE, "角色正在使用中，无法删除");
   }
   ```

## 解决方案

### 修改前端代码

**文件：** `frontend/src/views/RoleManagement/index.vue`

```typescript
const handleDelete = async (row: RoleInfo) => {
  try {
    await ElMessageBox.confirm(
      `确认要删除角色 "${row.name}" 吗？删除后不可恢复！`,
      '删除角色',
      {
        confirmButtonText: '确认',
        cancelButtonText: '取消',
        type: 'warning'
      }
    )

    await deleteRole(row.id)
    ElMessage.success('删除成功')
    loadRoles()
  } catch (error: any) {
    // 点击取消时，error 是 'cancel' 字符串
    if (error === 'cancel') {
      return
    }

    // 显示后端返回的具体错误信息
    const errorMessage = error?.message || '删除失败'
    ElMessage.error(errorMessage)
  }
}
```

### 修改要点

1. **区分取消和错误**
   - 用户点击取消：`error === 'cancel'`
   - 真正的错误：`error` 是 Error 对象

2. **提取错误信息**
   - 使用 `error?.message` 获取错误信息
   - 如果没有则显示默认的"删除失败"

## 修复效果

### 修改前
- ❌ 删除失败
- 用户不知道具体原因

### 修改后

**场景1：删除系统内置角色**
- ✅ 系统内置角色不能删除

**场景2：删除有用户使用的角色**
- ✅ 角色正在使用中，无法删除

**场景3：删除不存在的角色**
- ✅ 角色不存在

**场景4：正常删除**
- ✅ 删除成功

## 用户体验优化

1. **明确提示**：用户知道为什么不能删除
2. **可操作**：根据错误提示采取相应措施
   - 系统内置角色：不可删除
   - 有用户使用：先将用户分配到其他角色
3. **减少困惑**：不再出现通用的"删除失败"提示
