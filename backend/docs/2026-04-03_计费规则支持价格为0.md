# 2026-04-03 计费规则支持价格为0

## 修改日期
2026-04-03

## 修改内容
系统设置 -> 计费规则中，支持设置规则价格为0元，用于免费时段或促销活动。

## 修改的文件

### `frontend/src/views/Settings/index.vue`

**修改点 1：价格输入框最小值（第 84 行）**

修改前：
```vue
<el-input-number
  v-model="rule.price"
  :min="0.01"
  :max="999.99"
  :precision="2"
  :step="0.5"
  placeholder="价格"
  style="width: 120px"
/>
```

修改后：
```vue
<el-input-number
  v-model="rule.price"
  :min="0"
  :max="999.99"
  :precision="2"
  :step="0.5"
  placeholder="价格"
  style="width: 120px"
/>
```

**修改点 2：保存验证逻辑（第 634-637 行）**

修改前：
```typescript
if (!rule.price || rule.price < 0.01) {
  ElMessage.warning(`${channel.channelName}的规则价格必须大于0元`)
  return
}
```

修改后：
```typescript
if (rule.price == null || rule.price < 0) {
  ElMessage.warning(`${channel.channelName}的规则价格不能为负数`)
  return
}
```

## 使用场景

支持价格为0元可以用于以下场景：

1. **免费时段**
   - 开业前30分钟免费
   - 特定时段促销（如下午2点-4点免费）

2. **会员特权**
   - 金牌会员首次到店免费
   - VIP会员特定时段免费

3. **优惠活动**
   - 新用户体验时段
   - 节日促销时段

4. **测试规则**
   - 用于测试计费系统
   - 演示环境

## 验证规则

修改后的验证规则：
- ✅ 允许价格为 0 元
- ✅ 允许价格为正数（0.01, 0.5, 1 等）
- ❌ 不允许价格为负数
- ❌ 不允许价格为空（null/undefined）

## 注意事项

1. **精度保持**：价格字段仍保持2位小数精度
2. **步长调整**：步长仍为0.5，方便快速输入常用价格
3. **最大值限制**：最大值仍为999.99元
4. **向后兼容**：现有规则不受影响

## 测试要点

1. **输入0元**：验证可以成功保存
2. **输入-1元**：验证提示"价格不能为负数"
3. **清空价格**：验证提示"价格不能为负数"
4. **输入0.01元**：验证可以成功保存
5. **输入正常价格**：验证功能正常
