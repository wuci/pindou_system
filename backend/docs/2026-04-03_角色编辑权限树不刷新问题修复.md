# 2026-04-03 角色编辑权限树不刷新问题修复

## 修改日期
2026-04-03

## 问题描述

在角色管理页面，当点击编辑一个角色后，如果再次点击其他角色的编辑按钮，权限配置树仍然显示上一次打开的角色权限数据，没有更新。

### 问题现象

1. 点击角色A的编辑按钮，权限树正确显示角色A的权限
2. 关闭对话框，点击角色B的编辑按钮
3. 权限树仍然显示角色A的权限，没有更新为角色B的权限

## 问题原因

**文件**：`frontend/src/components/PermissionTree.vue`

**问题代码**：
```vue
<el-tree
  ref="treeRef"
  :data="permissionTree"
  :props="treeProps"
  show-checkbox
  node-key="permissionKey"
  :default-checked-keys="modelValue"
  @check="handleCheck"
  :loading="loading"
>
```

**原因分析**：
1. `el-tree` 组件的 `:default-checked-keys` 属性只在**初始化时生效**
2. 当 `modelValue` 变化时，`el-tree` 不会自动更新选中状态
3. 角色管理页面使用 `v-model="formData.permissions"` 绑定权限树
4. 当编辑不同角色时，`formData.permissions` 会变化，但权限树的选中状态没有同步更新

**相关代码**（角色管理页面）：
```typescript
// 编辑角色
const handleEdit = (row: RoleInfo) => {
  dialogTitle.value = '编辑角色'
  isEdit.value = true
  Object.assign(formData, {
    id: row.id,
    name: row.name,
    code: row.code,
    permissions: [...(row.permissions || [])],  // permissions 会变化
    sort: row.sort,
    status: row.status,
    description: row.description
  })
  dialogVisible.value = true
}
```

## 修复方案

在 `PermissionTree` 组件中添加对 `modelValue` 的监听，当 `modelValue` 变化时，使用 `setCheckedKeys` 方法更新树的选中状态。

**修改内容**：
```typescript
// 添加 watch 监听 modelValue 变化
watch(() => props.modelValue, (newVal) => {
  if (treeRef.value && newVal) {
    treeRef.value.setCheckedKeys(newVal)
  }
}, { immediate: false })
```

**完整代码**：
```typescript
const handleCheck = () => {
  if (!treeRef.value) return
  const checkedKeys = treeRef.value.getCheckedKeys()
  emit('update:modelValue', checkedKeys as string[])
}

// 监听 modelValue 变化，同步更新树的选中状态
watch(() => props.modelValue, (newVal) => {
  if (treeRef.value && newVal) {
    treeRef.value.setCheckedKeys(newVal)
  }
}, { immediate: false })

// 组件挂载时加载数据
onMounted(() => {
  loadPermissionTree()
})
```

## 技术说明

### Element Plus Tree 组件的 API

- `default-checked-keys`: 只在初始化时设置默认选中的节点
- `setCheckedKeys(keys)`: 方法，可以动态设置选中的节点

### 为什么使用 watch 而不是 computed

- `watch` 可以在值变化时执行副作用（调用方法）
- `computed` 用于派生新值，不适用于执行副作用
- `{ immediate: false }` 确保只在值变化时执行，不在初始化时执行

## 验证方法

1. 在角色管理页面点击角色A的编辑按钮
2. 查看权限配置树的选中状态
3. 关闭对话框
4. 点击角色B的编辑按钮
5. 权限配置树应正确显示角色B的权限
