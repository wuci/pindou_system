# 2026-04-03 侧边栏菜单权限控制修复

## 修改日期
2026-04-03

## 问题描述

1. **菜单显示问题**：会员管理和会员等级菜单没有添加权限控制，导致所有登录用户都能看到
2. **路由跳转问题**：已登录用户访问无权限页面时，错误地跳转到登录页

### 问题现象

- 店长账号没有配置 `member:level` 权限
- 但侧边栏仍然显示"会员等级"菜单
- 点击后因为权限检查失败，被路由守卫跳转到登录页（不符合预期）

## 问题原因

### 1. 侧边栏菜单缺少权限控制

**文件**：`frontend/src/layouts/DefaultLayout.vue`

**问题代码**（第 75-82 行）：
```vue
<!-- 没有权限控制，所有用户都能看到 -->
<el-menu-item index="/members">
  <el-icon><User /></el-icon>
  <span>会员管理</span>
</el-menu-item>
<el-menu-item index="/member-levels">
  <el-icon><Star /></el-icon>
  <span>会员等级</span>
</el-menu-item>
```

### 2. permissions computed 缺少权限定义

**问题代码**（第 127-132 行）：
```typescript
const permissions = computed(() => ({
  canViewUsers: userStore.hasPermission('user:view'),
  canManageRoles: userStore.hasPermission('role:manage'),
  canViewLogs: userStore.hasPermission('log:view'),
  canConfigSystem: userStore.hasPermission('system:config')
  // ❌ 缺少会员相关权限
}))
```

### 3. 路由守卫处理不当

**文件**：`frontend/src/router/index.ts`

**问题代码**（第 173-180 行）：
```typescript
if (to.meta.permission) {
  if (!userStore.hasPermission(to.meta.permission as string)) {
    // ❌ 已登录用户无权限时，仍然跳转到登录页
    next({ path: '/login' })
    return
  }
}
```

## 修复方案

### 1. 添加会员相关权限检查

**修改文件**：`frontend/src/layouts/DefaultLayout.vue`

**修改内容 1**（第 127-133 行）：
```typescript
const permissions = computed(() => ({
  canViewUsers: userStore.hasPermission('user:view'),
  canManageRoles: userStore.hasPermission('role:manage'),
  canViewLogs: userStore.hasPermission('log:view'),
  canViewMembers: userStore.hasPermission('member:view'),        // ✅ 新增
  canManageMemberLevels: userStore.hasPermission('member:level'), // ✅ 新增
  canConfigSystem: userStore.hasPermission('system:config')
}))
```

**修改内容 2**（第 75-82 行）：
```vue
<!-- ✅ 添加权限控制 -->
<el-menu-item index="/members" v-if="permissions.canViewMembers">
  <el-icon><User /></el-icon>
  <span>会员管理</span>
</el-menu-item>
<el-menu-item index="/member-levels" v-if="permissions.canManageMemberLevels">
  <el-icon><Star /></el-icon>
  <span>会员等级</span>
</el-menu-item>
```

### 2. 修复路由守卫逻辑

**修改文件**：`frontend/src/router/index.ts`

**修改内容 1**（添加导入）：
```typescript
import { createRouter, createWebHistory, RouteRecordRaw } from 'vue-router'
import { useUserStore } from '@/stores/user'
import { ElMessage } from 'element-plus'  // ✅ 新增
```

**修改内容 2**（第 173-182 行）：
```typescript
// 检查权限（只有已登录用户才检查权限）
if (to.meta.permission) {
  if (!userStore.hasPermission(to.meta.permission as string)) {
    console.log('无权限，重定向到首页')
    // ✅ 已登录用户无权限时，提示并跳转到首页
    ElMessage.warning('您没有访问该页面的权限')
    next({ path: '/dashboard' })
    return
  }
}
```

## 修复效果

### 修复前

| 用户权限 | 菜单显示 | 点击行为 |
|----------|----------|----------|
| 无 `member:level` | ✅ 显示 | 跳转到登录页 ❌ |
| 有 `member:level` | ✅ 显示 | 正常访问 ✅ |

### 修复后

| 用户权限 | 菜单显示 | 点击行为 |
|----------|----------|----------|
| 无 `member:level` | ❌ 不显示 | - |
| 有 `member:level` | ✅ 显示 | 正常访问 ✅ |

### 直接访问无权限页面

| 修复前 | 修复后 |
|--------|--------|
| 跳转到登录页 | 跳转到首页 + 提示"您没有访问该页面的权限" |

## 权限对照表

| 菜单项 | 路由 | 权限标识 | computed 属性 |
|--------|------|----------|---------------|
| 用户管理 | /users | `user:view` | canViewUsers |
| 角色管理 | /roles | `role:manage` | canManageRoles |
| 操作日志 | /logs | `log:view` | canViewLogs |
| 会员管理 | /members | `member:view` | canViewMembers |
| 会员等级 | /member-levels | `member:level` | canManageMemberLevels |
| 系统设置 | /settings | `system:config` | canConfigSystem |

2. **直接访问**：
   - 登录店长账号，直接在浏览器访问 `/member-levels`
   - 验证是否跳转到首页并显示权限提示

3. **权限切换**：
   - 给角色添加/移除 `member:level` 权限
   - 刷新页面，验证菜单显示/隐藏
