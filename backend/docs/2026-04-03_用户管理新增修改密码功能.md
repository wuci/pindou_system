# 用户管理新增修改密码功能

**日期：** 2026-04-03
**修改人：** wuci
**功能描述：** 在用户管理中添加"修改密码"功能，允许管理员自定义设置用户的新密码

## 功能说明

### 原有功能
- **重置密码**：将用户密码重置为默认密码 `123456`

### 新增功能
- **修改密码**：管理员可以自定义设置用户的新密码，需要输入并确认新密码

## 修改内容

### 1. 前端API层（frontend/src/api/user.ts）

添加了修改密码的接口定义：

```typescript
/**
 * 修改密码参数
 */
export interface ChangePasswordParams {
  userId: string
  newPassword: string
}

/**
 * 修改密码
 */
export const changePassword = (data: ChangePasswordParams) => {
  return http.put(`/users/${data.userId}/reset-password`, { password: data.newPassword })
}
```

### 2. 用户管理页面（frontend/src/views/UserManagement/index.vue）

#### 核心处理函数

1. **打开修改密码弹窗**
```typescript
const handleChangePassword = (row: UserInfo) => {
  passwordFormData.userId = row.id
  passwordFormData.username = row.username
  passwordFormData.newPassword = ''
  passwordFormData.confirmPassword = ''
  passwordDialogVisible.value = true
}
```

2. **提交修改密码**
```typescript
const handleChangePasswordSubmit = async () => {
  if (!passwordFormRef.value) return

  try {
    await passwordFormRef.value.validate()
    passwordSubmitting.value = true

    await changePassword({
      userId: passwordFormData.userId,
      newPassword: passwordFormData.newPassword
    })

    ElMessage.success('修改密码成功')
    passwordDialogVisible.value = false
  } catch (error) {
    if (error !== false) {
      ElMessage.error('修改密码失败')
    }
  } finally {
    passwordSubmitting.value = false
  }
}
```

3. **自定义验证：确认密码**
```typescript
const validateConfirmPassword = (rule: any, value: any, callback: any) => {
  if (value === '') {
    callback(new Error('请再次输入密码'))
  } else if (value !== passwordFormData.newPassword) {
    callback(new Error('两次输入密码不一致'))
  } else {
    callback()
  }
}
```

## 功能对比

| 功能 | 修改前 | 修改后 |
|-----|--------|--------|
| 重置密码 | 密码重置为 123456 | 密码重置为 123456 |
| 修改密码 | ❌ 不存在 | ✅ 可自定义新密码 |

## 表单验证规则

### 新密码
- 必填项
- 长度：6-20 个字符

### 确认密码
- 必填项
- 必须与新密码一致

## 权限控制

使用现有的 `user:resetPassword` 权限控制"修改密码"和"重置密码"按钮的显示。

## 用户体验优化

1. **操作列宽度调整**：从 220px 增加到 280px，以容纳新增的按钮
2. **按钮样式区分**：
   - 修改密码：warning 样式（橙色）
   - 重置密码：info 样式（灰色）
3. **表单验证**：实时验证两次密码输入是否一致
4. **加载状态**：提交时显示加载状态，防止重复提交

## 使用场景

### 重置密码
- 用户忘记密码时的快速恢复
- 将密码重置为默认值
- 无需知道原密码

### 修改密码
- 管理员主动为用户设置新密码
- 需要设置特定的强密码
- 定期更换用户密码的安全管理
