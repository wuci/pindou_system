# 2026-04-03 权限树为空问题修复

## 修改日期
2026-04-03

## 问题描述

在角色管理页面点击编辑时，权限配置列表显示为空。

## 问题原因

**文件**：`backend/pdsystem-service/src/main/java/com/pindou/timer/service/impl/PermissionServiceImpl.java`

**问题代码**（第 51 行）：
```java
return buildTree(responseList, "root");
```

**原因分析**：
1. `buildTree` 方法使用 `"root"` 作为顶级父节点ID来查找根节点
2. 但在 SQL 初始化脚本中，所有顶级权限的 `parent_id` 定义为 `'0'`
3. 导致 `buildTree` 方法找不到任何根节点，返回空列表

**SQL 定义**：
```sql
INSERT INTO sys_permission (id, parent_id, ...) VALUES
('dashboard', '0', 'dashboard:view', '工作台', ...);
--                 ^^^^
--         parent_id 是 '0'，不是 'root'
```

## 修复方案

将 `getPermissionTree()` 方法中的父节点ID从 `"root"` 改为 `"0"`。

**修改内容**：
```java
// 修改前
return buildTree(responseList, "root");

// 修改后
return buildTree(responseList, "0");
```

## 验证方法

1. 重新编译后端项目
2. 重启后端服务
3. 在角色管理页面点击编辑
4. 权限配置树应正常显示所有权限
