```typescript
// 获取支付方式标签
const getPaymentMethodLabel = (method: string) => {
  const labels: Record<string, string> = {
    offline: '线下支付',
    online: '线上支付',
    balance: '会员余额',
    combined: '组合支付'
  }
  return labels[method] || method
}

// 获取支付方式标签类型
const getPaymentMethodTagType = (method: string) => {
  const types: Record<string, string> = {
    offline: '',
    online: 'primary',
    balance: 'success',
    combined: 'warning'
  }
  return types[method] || ''
}
```

## 功能说明

### 字段说明

1. **paymentMethod（支付方式）**
   - `offline`：线下支付（无颜色标签）
   - `online`：线上支付（蓝色标签）
   - `balance`：会员余额（绿色标签）
   - `combined`：组合支付（黄色标签）

2. **balanceAmount（余额支付金额）**
   - 仅在余额支付或组合支付时显示
   - 绿色文字，加粗显示
   - 显示实际从余额扣除的金额

3. **otherPaymentAmount（其他方式支付金额）**
   - 仅在组合支付时显示
   - 红色文字，加粗显示
   - 显示需要线下支付的金额

4. **paidAt（支付时间）**
   - 仅在已支付（completed 状态）时显示
   - 格式化为日期时间格式
   - 从时间信息移至支付方式部分

### 显示逻辑

1. **整个支付方式区域**：仅在 `detail.paymentMethod` 存在时显示
2. **余额支付金额**：仅在 `detail.balanceAmount > 0` 时显示
3. **线下支付金额**：仅在 `detail.otherPaymentAmount > 0` 时显示
4. **支付时间**：仅在 `detail.paidAt` 存在时显示

### 样式特点

1. **标签颜色区分**：不同支付方式使用不同颜色标签
   - 会员余额：绿色（success）
   - 组合支付：黄色（warning）
   - 线上支付：蓝色（primary）
   - 线下支付：无颜色（默认）

2. **金额颜色区分**：
   - 余额支付：绿色（#67c23a）
   - 线下支付：红色（#f56c6c）
   - 加粗显示，醒目清晰

3. **布局设计**：
   - 使用 info-grid 布局，两列显示
   - 标签在上，值在下
   - 间距合理，清晰美观

## 数据流程

1. **订单创建/更新**：
   - TableServiceImpl 在操作订单时设置支付方式字段
   - 设置 `paymentMethod`、`balanceAmount`、`otherPaymentAmount`
   - 结账时设置 `paidAt` 时间戳

2. **查询订单详情**：
   - OrderServiceImpl.getOrderDetail() 查询订单
   - convertToDetailResponse() 转换为响应对象
   - 设置支付方式相关字段到响应对象

3. **前端显示**：
   - OrderDetailDrawer 组件接收订单详情
   - 根据字段值判断显示哪些信息
   - 应用相应的样式和颜色
