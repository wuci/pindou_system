## 权限设计说明

### 1. 模块划分原则

根据实际业务功能将权限分为10个模块：

- **工作台**：数据概览，使用 `table:view` 权限
- **桌台管理**：核心业务，包含完整的计时流程权限
- **订单管理**：订单相关操作
- **数据统计**：报表查看
- **用户管理**：系统用户管理
- **角色管理**：权限管理，使用统一的 `role:manage` 权限
- **操作日志**：日志查看
- **会员管理**：会员信息及充值
- **会员等级**：会员等级配置
- **系统设置**：计费规则等系统配置

### 2. 权限命名规范

遵循 `{模块}:{操作}` 的命名格式：

```
{模块}:view          # 查看
{模块}:create        # 新增
{模块}:update        # 编辑
{模块}:delete        # 删除
{模块}:{具体操作}     # 特定操作（如 start, pause, resume, end）
```

**特例**：
- `role:manage` - 角色管理统一权限
- `member:level` - 会员等级管理
- `system:config` - 系统配置
- `table:view` - 同时用于工作台和桌台管理

### 3. 图标使用规范

使用有效的 Element Plus 图标：

| 功能 | 图标 | 图标类名 |
|------|------|----------|
| 工作台 | 🎚️ | Odometer |

| 操作日志 | 📒 | Notebook |
| 会员等级 | ⭐ | Star |
| 系统设置 | 🔧 | Tools |

## 数据迁移说明

如果数据库中已有角色数据，可能需要进行权限标识的迁移：

### 旧权限标识 → 新权限标识

| 旧标识 | 新标识 | 说明 |
|--------|--------|------|
| `statistics:view` | `report:view` | 数据统计 |
| `config:view` | `system:config` | 系统设置 |
| `config:update` | `system:config` | 系统设置 |
| `role:view` | `role:manage` | 角色管理 |
| `role:create` | `role:manage` | 角色管理 |
| `role:update` | `role:manage` | 角色管理 |
| `role:delete` | `role:manage` | 角色管理 |

**注意**：如果旧数据使用了已删除的权限标识，需要更新为新标识。
