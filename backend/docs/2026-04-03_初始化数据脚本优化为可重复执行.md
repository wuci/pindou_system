# 初始化数据脚本优化为可重复执行

**日期：** 2026-04-03
**操作人：** wuci
**文件：** backend/sql/20260403/02_初始化数据脚本.sql

## 优化内容

### 1. 支持重复执行

采用先清空表再插入数据的方式，确保脚本可以重复执行并每次都获得全新的初始数据。

**执行流程：**
```sql
-- 1. 禁用外键检查
SET FOREIGN_KEY_CHECKS = 0;

-- 2. 清空所有表数据（按依赖关系逆序）
TRUNCATE TABLE `biz_consumption_record`;
TRUNCATE TABLE `biz_recharge_record`;
...

-- 3. 恢复外键检查
SET FOREIGN_KEY_CHECKS = 1;

-- 4. 插入初始数据
INSERT INTO `sys_config` (...) VALUES (...);
...
```

### 2. 清空表顺序

按照外键依赖关系的逆序清空表，避免外键约束错误：

```
biz_consumption_record (依赖 biz_member, biz_order)
biz_recharge_record (依赖 biz_member)
biz_order (依赖 biz_table)
biz_table (依赖 biz_table_category)
biz_member (依赖 biz_member_level)
biz_table_layout_config (依赖 biz_table_category)
biz_table_category
biz_member_level
sys_operation_log
sys_user (依赖 sys_role)
sys_role
sys_permission
sys_config
```

### 3. 按表分组整理

将INSERT语句按表进行分组，并添加清晰的注释说明：

1. 系统配置表（5条）
2. 权限表（48条）
3. 角色表（4条）
4. 用户表（3条）
5. 桌台分类表（2条）
6. 会员等级配置表（4条）
7. 桌台表（20条）

### 4. 清理历史数据

将桌台表中的测试数据（订单ID、开始时间等）清理为初始状态，用户表的登录信息也重置：
```sql
-- 桌台表：清理订单相关数据
current_order_id = NULL
start_time = NULL
last_login_at = NULL
last_login_ip = NULL

-- 用户表：重置登录信息
last_login_at = NULL
last_login_ip = NULL
```

### 5. 添加ID字段

为自增ID表（biz_table_category、biz_member_level）添加明确的ID值，确保数据一致性：
```sql
INSERT INTO `biz_table_category` (id, name, ...) VALUES (1, '大厅', ...);
INSERT INTO `biz_member_level` (id, name, ...) VALUES (1, '豆豆萌新', ...);
```

### 6. 添加执行完成提示

在脚本结尾添加数据统计查询，方便验证脚本执行结果。

## 脚本特点

✅ **可重复执行**：先清空表再插入，每次都是全新数据
✅ **结构清晰**：按表分组，注释完整
✅ **数据干净**：移除测试数据，保持初始状态
✅ **易于验证**：执行完成后自动统计数据量
✅ **外键安全**：自动禁用/恢复外键检查
