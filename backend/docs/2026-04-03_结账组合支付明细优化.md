# 2026-04-03 结账组合支付明细优化

## 修改日期
2026-04-03

## 修改内容
优化结账时的支付方式明细显示，当选择组合支付时，清楚地展示会员余额支付和线下支付的具体金额，让用户对支付明细一目了然。

## 问题描述
之前的结账账单中，虽然显示了支付方式，但对于组合支付，显示的是"其他支付"，不够清晰。用户无法清楚地知道：
- 会员余额支付了多少钱
- 线下支付了多少钱
- 具体的支付拆分明细

## 显示效果

### 场景 1：组合支付

当订单金额为 54 元，会员余额 30 元时：

```
支付方式
组合支付
───────────
支付明细

💰 会员余额支付          -¥30.00
🏪 线下支付              -¥24.00
```

- 会员余额支付：绿色背景，绿色文字，左侧绿色边框
- 线下支付：红色背景，红色文字，左侧红色边框
- 图标 + 文字，更加直观

### 场景 2：会员余额支付

当订单金额为 35 元，会员余额充足时：

```
支付方式
会员余额
───────────

💰 会员余额支付          -¥35.00
```

### 场景 3：线下支付/线上支付

```
支付方式
线下支付
```

不显示支付明细（因为没有拆分）

## 样式特点

1. **图标化展示**：
   - 会员余额支付：钱包图标（WalletFilled）
   - 线下支付：店铺图标（ShopFilled）

2. **颜色区分**：
   - 会员余额：绿色主题（#67c23a）
   - 线下支付：红色主题（#f56c6c）

3. **背景色**：
   - 会员余额：浅绿色背景（#e7f7e9）
   - 线下支付：浅红色背景（#fef0f0）

4. **边框强调**：
   - 左侧 3px 彩色边框，增强视觉识别

5. **布局清晰**：
   - 支付明细标题：灰色小字
   - 每个支付项独立卡片
   - 间距合理，不拥挤

## 优势

1. ✅ **清晰直观**：用户一眼就能看到支付明细
2. ✅ **信息完整**：清楚显示每种支付方式的金额
3. ✅ **视觉美观**：图标+颜色+边框，现代美观
4. ✅ **易于理解**：文字描述清晰，"会员余额支付"和"线下支付"一目了然
5. ✅ **专业规范**：符合财务对账的显示要求

## 数据来源

后端 `BillResponse` 已包含以下字段：
- `paymentMethod`：支付方式
- `balanceAmount`：余额支付金额
- `otherPaymentAmount`：其他方式支付金额（线下支付）

后端 `TableServiceImpl.getBill()` 方法已正确设置这些字段：
```java
if (order.getPaymentMethod() != null) {
    response.setPaymentMethod(order.getPaymentMethod());
    response.setBalanceAmount(order.getBalanceAmount() != null ?
        order.getBalanceAmount().doubleValue() : null);
    response.setOtherPaymentAmount(order.getOtherPaymentAmount() != null ?
        order.getOtherPaymentAmount().doubleValue() : null);
}
```

## 用户操作流程

### 场景：结账查看组合支付明细

1. 用户点击"结账"按钮
2. 打开账单对话框
3. 查看费用明细
4. 看到"支付方式"部分
5. **组合支付时**：
   - 显示"组合支付"标题
   - 显示"支付明细"小标题
   - 显示"💰 会员余额支付 -¥30.00"（绿色）
   - 显示"🏪 线下支付 -¥24.00"（红色）
6. 用户清楚知道支付拆分明细
7. 点击"确认结账"
