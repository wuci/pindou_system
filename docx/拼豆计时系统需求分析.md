## 一、项目概述

### 1.1 项目背景
拼豆店提供手工制作体验服务，顾客按桌台入座，按使用时长收费。需要一个Web端管理系统，支持多店员登录、角色权限控制、桌台计时计费、订单管理、操作日志记录和超时提醒功能。

### 1.2 技术架构
- **前端**：Vue 3 + Composition API + Element Plus + Pinia + Vue Router
- **后端**：RESTful API（需对接）
- **认证方式**：账号密码登录 + Token认证
- **登录策略**：单点登录（同一账号同时只能一个设备在线）
- **数据存储**：后端数据库存储

## 三、功能模块详情

### 模块1：登录认证

| 功能点 | 描述 |
|--------|------|
| 登录页面 | 账号、密码输入框，登录按钮，记住密码选项 |
| 账号密码登录 | 提交到后端验证，验证成功返回Token |
| Token认证 | 登录成功存储Token，后续请求携带在请求头中 |
| 单点登录 | 同一账号同时只能一个设备在线，新登录踢掉旧会话 |
| Token过期处理 | Token过期时自动跳转登录页 |
| 记住密码 | 可选，记住账号密码（前端加密存储） |
| 手动登出 | 点击退出，清除Token，跳转登录页 |

### 模块6：超时提醒功能

#### 6.1 提醒触发规则

| 提醒类型 | 触发条件 | 说明 |
|----------|----------|------|
| 即将到期提醒 | 剩余时间 ≤ 提醒阈值 | 预设时长结束前N分钟触发 |
| 已超时提醒 | 已用时间 > 预设时长 | 超过预设时长后持续提醒 |

> 提醒阈值可管理员自定义（如3分钟、5分钟、10分钟）

#### 6.2 提醒方式

| 提醒方式 | 即将到期 | 已超时 | 说明 |
|----------|----------|--------|------|
| 卡片颜色 | 🟡 黄色渐变背景 | 🔴 红色渐变背景 | 状态色区分 |
| 卡片动画 | 闪烁边框 | 脉冲呼吸效果 | 吸引注意力 |
| 剩余/超时显示 | 显示倒计时（剩余 X分X秒） | 显示超时时间（超时 X分X秒） | 卡片内展示 |
| 顶部弹窗通知 | 3秒自动消失 | 5秒自动消失 | 全局消息提醒 |
| 声音提醒 | 轻提示音（1次） | 重复提示音（每30秒） | 可配置开关 |
| 提醒汇总面板 | 集中展示所有待处理提醒 | 集中展示所有待处理提醒 | 侧边栏展示 |

#### 6.3 提醒汇总面板

在界面右侧增加提醒汇总面板，集中展示所有需要关注的桌台：

- 点击"去处理"弹出操作菜单（续时/结账）
- 点击"忽略"后该次提醒不再弹出弹窗，但卡片颜色保留

#### 6.4 提醒配置项

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 提醒阈值 | 提前多少分钟提醒 | 5分钟 |
| 声音提醒 | 是否开启声音 | 开启 |
| 重复提醒间隔 | 超时后每隔多久提醒一次 | 30秒 |
| 弹窗自动关闭时间（即将） | 通知弹窗显示时长 | 3秒 |
| 弹窗自动关闭时间（超时） | 通知弹窗显示时长 | 5秒 |

九、操作流程
9.1 登录流程
打开系统 → 登录页面 → 输入账号密码 → 验证通过 → 跳转首页（根据权限展示功能）

9.2 计时操作流程
店员点击空闲桌台
       ↓
权限校验：是否有 table:start 权限
       ↓ 有权限
弹窗设置时长（预设/自定义/不设时长）
       ↓
确认开始计时
       ↓
桌台状态变更为"使用中"，计时开始
9.3 提醒处理流程
计时中，剩余时间 ≤ 提醒阈值
       ↓
触发即将到期提醒：
  - 桌台卡片变黄色+闪烁边框
  - 显示倒计时
